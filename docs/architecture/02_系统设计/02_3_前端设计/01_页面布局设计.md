## 10.1 项目概述

### 10.1.1 设计目标

GraphHire 前端分为三个独立的用户界面，通过统一的架构设计实现代码复用与安全隔离：

| 端 | 用户群体 | URL 前缀 | 部署 |
|----|---------|----------|------|
| 个人用户端 | 求职者 | `/person/*` | 公网 |
| 企业用户端 | 招聘方 | `/company/*` | 公网 |
| 管理后台 | 平台管理员 | `/admin/*` | 内网/白名单 |

**核心设计原则：**
- 个人端与企业端共享同一布局和大量组件，仅数据层和业务逻辑不同
- 管理后台独立部署，交互模式与 C 端完全不同
- C 端统一登录，A 端独立登录，JWT Token 完全隔离

### 10.1.2 技术栈

## 10.4 登录认证架构

### 10.4.1 三端登录关系

```
C 端统一登录                    A 端独立登录
┌──────────────────┐          ┌──────────────────┐
│   /login         │          │  /admin/login    │
│  (role=person)  ─┼─ JWT ─→  │                  │
│  (role=company)  │          │  独立 session    │
└──────────────────┘          └────────┬─────────┘
                                        │ JWT role=ADMIN
                                        ↓
                              /admin/dashboard
```

| 登录入口 | URL | 角色 | Token key |
|----------|-----|------|-----------|
| C 端统一登录 | `/login?role=person` | `PERSON` | `token` |
| C 端统一登录 | `/login?role=company` | `COMPANY` | `token` |
| A 端独立登录 | `/admin/login` | `ADMIN` | `admin_token` |

### 10.4.2 JWT Token 设计

**C 端 Token Payload：**
```json
{
  "sub": "user_id",
  "role": "PERSON" | "COMPANY",
  "exp": 1234567890
}
```

**A 端 Token Payload：**
```json
{
  "sub": "admin_id",
  "role": "ADMIN",
  "exp": 1234567890
}
```

### 10.4.3 登录流程

**C 端统一登录：**
1. 访问 `/login`，默认 role=`person`，页面展示"我是求职者"Tab
2. 用户选择"我是招聘方"Tab，URL 变为 `/login?role=company`
3. 登录成功后，后端返回 JWT，role 字段标识身份
4. 前端根据 role 跳转到 `/person/home` 或 `/company/home`
5. 后续所有请求携带 C 端 token，接口层根据 role 路由到对应数据

**A 端独立登录：**
1. 访问 `/admin/login`，独立登录页
2. 登录成功后，后端返回 admin_token
3. 跳转到 `/admin/dashboard`
4. A 端所有请求携带 admin_token，与 C 端 token 完全隔离

### 10.4.4 路由保护

```tsx
// middleware.ts — Next.js Middleware
export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl;

  // A 端路由保护
  if (pathname.startsWith("/admin") && !pathname.startsWith("/admin/login")) {
    const token = request.cookies.get("admin_token");
    if (!token) return NextResponse.redirect(new URL("/admin/login", request.url));
  }

  // C 端路由保护（person/company 通用）
  if (pathname.startsWith("/person") || pathname.startsWith("/company")) {
    const token = request.cookies.get("token");
    if (!token) return NextResponse.redirect(new URL("/login", request.url));
  }
}

export const config = {
  matcher: ["/person/:path*", "/company/:path*", "/admin/:path*"],
};
```

## 10.5 Next.js 新能力应用

### 10.5.1 Server Actions — 服务端数据操作

所有数据变更操作使用 Server Actions，替代传统的 API 路由调用，减少网络请求层级。

**适用场景：**

| 操作 | Server Action | 说明 |
|------|--------------|------|
| 简历上传 | `createResume` | 表单直接提交，自动处理文件上传 |
| 个人信息更新 | `updatePersonInfo` | 表单提交，无需 API 调用 |
| 职位发布 | `createJob` | 企业发布职位 |
| 管理员审核 | `authenticateCompany` | 企业认证审核 |
| 技能标签增删改 | `createSkill` / `updateSkill` / `deleteSkill` | 管理员操作 |

**代码示例：**

```tsx
// app/person/resume/actions.ts
"use server";

import { revalidatePath } from "next/cache";
import { redirect } from "next/navigation";

export async function createResume(formData: FormData) {
  const file = formData.get("file") as File;
  // 调用后端服务...
  const result = await resumeService.upload(file);

  // 清理缓存，触发 UI 更新
  revalidatePath("/person/resume");
  redirect("/person/resume");
}

// app/person/resume/page.tsx
import { createResume } from "./actions";

export default function ResumePage() {
  return (
    <form action={createResume}>
      <input type="file" name="file" accept=".pdf,.doc,.docx" />
      <button type="submit">上传简历</button>
    </form>
  );
}
```

**渐进增强：** 表单在 JS 禁用时仍可正常提交，服务端独立处理。

### 10.5.2 Parallel Routes + Intercepting Routes — 模态框路由拦截

候选人详情、企业信息等使用 Next.js 原生路由拦截，实现可分享的模态框。

**路由结构：**

```
app/(main)/company/match/
├── page.tsx                          # /company/match — 候选人列表页
├── @modal/
│   ├── (.)resume/[resumeId]/
│   │   └── page.tsx                 # 拦截路由：模态框显示
│   └── default.tsx                  # 无模态时渲染 null
└── resume/
    └── [resumeId]/
        └── page.tsx                  # 实际页面：完整详情页
```

**工作原理：**

```
访问 /company/match/resume/123
├── URL 可分享（复制链接给同事）
├── 浏览器历史正确（后退到列表）
└── 模态框覆盖在列表页上

访问 /company/match（无 ID 参数）
├── @modal/default.tsx → null
└── 仅显示候选人列表
```

**对应布局代码：**

```tsx
// app/(main)/company/match/layout.tsx
export default function MatchLayout({
  children,
  modal,
}: {
  children: React.ReactNode;
  modal: React.ReactNode;
}) {
  return (
    <div>
      {children}
      {modal}  {/* 模态框层 */}
    </div>
  );
}
```

**优势对比：**

| 特性 | 传统 Drawer | Intercepting Routes |
|------|------------|-------------------|
| URL 可分享 | ❌ | ✅ |
| SEO 友好 | ❌ | ✅ |
| 浏览器历史 | ❌ | ✅ |
| 刷新后状态 | 丢失 | 保持 |
| 链接预览 | 无 | 有 metadata |

### 10.5.3 Streaming + Suspense — 流式加载

页面内容按优先级流式传输，首屏快速响应，动态内容逐步加载。

**应用场景：**

```tsx
// app/person/home/page.tsx
import { Suspense } from "react";
import { RecommendedJobsSkeleton, StatsSkeleton } from "./loading";

export default function PersonHomePage() {
  return (
    <div>
      {/* 静态内容立即渲染 */}
      <WelcomeBanner />

      {/* 动态内容流式加载 */}
      <Suspense fallback={<StatsSkeleton />}>
        <StatsData />  {/* 从数据库获取 */}
      </Suspense>

      <Suspense fallback={<RecommendedJobsSkeleton />}>
        <RecommendedJobs />  {/* AI 匹配计算 */}
      </Suspense>

      <Suspense fallback={<GraphSkeleton />}>
        <MiniGraph />  {/* Neo4j 图谱查询 */}
      </Suspense>
    </div>
  );
}
```

**加载骨架屏：**

```tsx
// app/person/home/loading.tsx
export function StatsSkeleton() {
  return (
    <div className="grid grid-cols-4 gap-4">
      {[...Array(4)].map((_, i) => (
        <div key={i} className="animate-pulse h-24 bg-gray-200 rounded-lg" />
      ))}
    </div>
  );
}
```

### 10.5.4 Partial Prerendering (PPR) — 静态壳 + 动态核

页面由静态 HTML 壳 + 动态内容组成，首屏极速加载。

**适用页面：**

| 页面 | 静态部分 | 动态部分 |
|------|---------|---------|
| `/person/home` | 页面框架、欢迎语 | 推荐职位、匹配数、图谱预览 |
| `/company/match/:id` | 候选人骨架 | 匹配度评分、技能对比 |
| `/admin/dashboard` | 页面布局 | 统计数据、图表 |
| `/person/graph` | 页面框架 | 图谱节点数据 |

**实现方式：**

```tsx
// app/person/graph/page.tsx
import { Prerender } from "next";

export default function GraphPage() {
  return (
    <div>
      {/* 静态部分：立即渲染 */}
      <PageHeader title="个人能力图谱" />

      {/* 动态部分：运行时流式渲染 */}
      <Prerender fallback={<GraphSkeleton />}>
        <ForceGraph data={graphData} />  {/* Neo4j 查询 */}
      </Prerender>

      <Prerender fallback={<SkillsPanelSkeleton />}>
        <SkillsPanel skills={skills} />  {/* 数据库查询 */}
      </Prerender>
    </div>
  );
}
```

### 10.5.6 next/font — 字体优化

字体文件在构建时下载并自托管，消除布局偏移（CLS）。

**使用方式：**

```tsx
// app/layout.tsx
import { Inter } from "next/font/google";
import { Noto_Sans_SC } from "next/font/google";

const inter = Inter({ subsets: ["latin"] });
const notoSansSC = Noto_Sans_SC({ subsets: ["latin"], weight: ["400", "600"] });

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="zh-CN" className={`${inter.variable} ${notoSansSC.variable}`}>
      <body>{children}</body>
    </html>
```

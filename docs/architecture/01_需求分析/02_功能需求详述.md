# 功能需求详述

## 2.0 基本功能要求

本系统采用 **B/S架构（Browser/Server）** 开发，具体要求如下：

- **数据库**：采用 PostgreSQL 存储结构化数据
- **大模型服务**：支持本地大模型服务部署（Ollama）或调用云端大模型API
- **可视化界面**：具备 PC Web 响应式界面
- **文件存储**：支持 DOC、PDF 格式文档的存储与解析

---

## 2.1 双角色注册与登录

### 注册流程

**个人用户注册：**
1. 填写用户名（邮箱）、密码，确认密码
2. 系统发送邮箱验证码至注册邮箱
3. 输入验证码完成验证
4. 注册成功，自动登录

**企业用户注册：**
1. 填写企业名称、统一社会信用代码
2. 填写管理员账号（邮箱/用户名）、密码
3. 系统发送邮箱验证码至注册邮箱
4. 输入验证码完成验证
5. 注册成功，需等待管理员审核企业资质

### 登录流程

| 角色 | 登录方式 |
|------|----------|
| 个人用户 | 用户名/邮箱 + 密码 |
| 企业用户 | 管理员账号 + 密码 |
| 管理员 | 管理员专属账号 + 密码 |

**安全机制：**
- 连续5次登录失败，锁定账号15分钟
- 密码采用BCrypt加密存储
- JWT Token有效时长为2小时，支持 Refresh Token 续期

## 2.2 文档智能解析与结构化

### 支持格式
- DOC（.doc, .docx）
- PDF（.pdf）

### 解析内容

**从简历中提取：**

| 字段 | 说明 |
|------|------|
| 基本信息 | 姓名、性别、年龄、联系方式、学历 |
| 求职意向 | 意向职位、意向城市、薪资期望 |
| 工作经历 | 公司名称、职位、时间段、工作内容 |
| 教育经历 | 学校、专业、学历、时间段 |
| 技能标签 | 技术技能、软技能（由AI提取并归一化） |
| 项目经验 | 项目名称、角色、技术栈、项目描述 |

**从职位描述中提取：**

| 字段 | 说明 |
|------|------|
| 职位信息 | 职位名称、所属部门、招聘人数 |
| 工作地点 | 城市、区县、详细地址 |
| 薪资范围 | 最低薪资、最高薪资、薪资单位 |
| 技能要求 | 必须技能、优先技能（由AI提取并归一化） |
| 经验要求 | 工作年限要求、学历要求 |
| 岗位职责 | 核心职责列表 |
| 加分项 | 其他加分条件 |

### 解析流程

```
文档上传 → 文件类型校验 → 存储RustFS → 创建解析任务
    ↓
任务队列（RocketMQ） → AI服务消费 → Apache Tika文本提取
    ↓
大模型智能提取 → 技能标签生成 → 结构化数据入库PostgreSQL
    ↓
触发图谱构建 → Neo4j更新节点和关系
```

### 解析状态与流转

| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| 0 | 待解析 | 文档已上传，等待任务队列消费 |
| 1 | 解析中 | AI服务正在解析文档 |
| 2 | 解析成功 | 文档解析完成，数据已入库 |
| 3 | 解析失败 | 解析过程出错，可重试 |

### 解析异常处理

| 异常情况 | 处理方式 |
|----------|----------|
| 文件格式不支持 | 返回错误提示"仅支持DOC、PDF格式" |
| 文件过大（>10MB） | 返回错误提示"文件大小不能超过10MB" |
| 文件损坏无法读取 | 标记为解析失败，记录错误信息 |
| 大模型服务不可用 | 自动降级为关键词提取，标记解析结果置信度低 |
| 网络超时 | 重试3次，失败后标记状态为失败 |

### 解析结果置信度

| 置信度等级 | 分值范围 | 说明 |
|------------|----------|------|
| 高 | 0.8-1.0 | AI提取结果可靠，可直接使用 |
| 中 | 0.5-0.8 | 建议用户人工核对 |
| 低 | 0-0.5 | 提取结果仅供参考，需用户补充完善 |

## 2.3 能力图谱构建与可视化

### 个人能力图谱

- **节点类型**：技能、教育背景、工作经验、项目经历
- **关系类型**：掌握（技能熟练度）、经历（工作/项目经验时长）
- **展示方式**：力导向图 + 技能雷达图

### 职位能力图谱

- **节点类型**：必须技能、优先技能、软技能
- **关系类型**：要求程度（必须/优先）、关联技能
- **展示方式**：技能树 + 重要性权重标注

### 图谱能力指标

| 指标 | 说明 |
|------|------|
| 技能匹配度 | 个人技能与职位技能的重合度 |
| 经验匹配度 | 工作年限、职位级别的匹配程度 |
| 城市匹配度 | 意向城市与工作城市的符合度 |
| 综合匹配度 | 多维度加权计算的最终匹配分（0-100分） |

## 2.4 智能双向匹配与推荐

### 个人→职位推荐

1. 用户上传简历并完成解析
2. 系统提取其技能列表与意向条件
3. 从数据库查询技能相关的职位
4. 将数据发送给大模型计算匹配度得分
5. 返回排序后的职位推荐列表

### 企业→个人推荐

1. 企业上传职位描述并完成解析
2. 系统提取职位所需技能与要求
3. 从数据库查询具备相关技能的人才
4. 将数据发送给大模型计算匹配度得分
5. 返回排序后的候选人推荐列表

### 大模型匹配计算

大模型根据用户和职位的多维度信息，计算综合匹配度：

| 维度 | 权重 | 说明 |
|------|------|------|
| 技能匹配 | 50% | 核心技能的重合度与覆盖度 |
| 经验匹配 | 20% | 工作年限与职位级别的符合度 |
| 城市匹配 | 15% | 意向城市与工作城市的符合度 |
| 学历匹配 | 10% | 学历要求与实际学历的符合度 |
| 薪资匹配 | 5% | 期望薪资与职位薪资范围的符合度 |

### 大模型匹配优势

- **语义理解**：能识别"Spring Boot"与"Spring"是相关技能
- **智能归一化**：自动将不同表述归一化到统一标签
- **生成匹配理由**：AI生成个性化匹配分析报告
- **发现差距**：指出用户技能与职位要求的差距

### 匹配结果展示

- 总体匹配分（0-100分）
- 各维度得分雷达图
- 技能对比列表（已具备/未具备/弱项）
- 匹配依据说明（AI生成）

## 2.7 数据校验规则

### 用户输入校验

| 字段 | 校验规则 |
|------|----------|
| 用户名/邮箱 | 邮箱格式校验，唯一性检查 |
| 密码 | 8-20位，必须包含字母和数字 |
| 手机号 | 11位数字，以1开头 |
| 统一社会信用代码 | 18位代码，格式校验 |

| 薪资范围 | 最低薪资 ≤ 最高薪资 |

### 文档上传校验

| 校验项 | 规则 |
|--------|------|
| 文件格式 | 仅支持 .doc、.docx、.pdf |
| 文件大小 | 单文件不超过 10MB |
| 文件数量 | 个人用户最多上传5份简历 |

## 2.9 安全要求

| 安全项 | 要求 |
|--------|------|
| 密码存储 | BCrypt加密，不可逆 |
| 传输安全 | HTTPS全链路加密 |
| 认证机制 | JWT Token + Refresh Token |
| 权限控制 | 基于用户类型的访问控制（user_type + post） |
| 敏感操作 | 需二次验证（如删除数据） |

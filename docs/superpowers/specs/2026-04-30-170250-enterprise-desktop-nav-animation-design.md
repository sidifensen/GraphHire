# 企业端桌面菜单切换动效设计

## 目标

为企业端桌面顶部菜单增加更丝滑的切换反馈，提升菜单在不同页面之间跳转时的连续感，不改变现有信息架构与路由行为。

## 范围

- 仅调整企业端桌面 `TopNav` 的菜单视觉动效。
- 保持移动端菜单行为不变。
- 不修改路由映射、权限逻辑、页面内容布局。

## 方案

1. 为桌面菜单增加“轨道容器”样式，提供轻微背景与内阴影，形成可感知的承载层。
2. 保留现有 `layoutId` 共享布局动画能力，将激活态背景改为更明确的“滑块”视觉。
3. 为菜单项增加 `hover` 和 `active` 的过渡（颜色、位移、缩放），让状态切换更连贯。
4. 对图标和文案同步添加短时过渡，避免滑块移动时内容突兀跳变。

## 测试策略

- 组件测试验证：桌面导航轨道与激活态指示器节点存在。
- 手工浏览器验证：在企业端桌面宽度下切换“工作台/职位/推荐/团队”，观察滑块跟随与过渡是否连续。

## 风险与回退

- 风险：过强动效可能造成视觉干扰。
- 缓解：控制位移和时长在轻量范围（约 150ms~280ms）。
- 回退：可仅保留共享 `layoutId` 滑块动画，移除 hover 位移与缩放类名。

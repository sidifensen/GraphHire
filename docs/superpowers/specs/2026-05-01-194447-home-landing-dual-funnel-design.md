# 首页双入口落地页重构设计（科技理性 + 品牌高端）

- 日期：2026-05-01 19:44:47
- 需求来源：用户要求首页落地页全量重构，双目标转化（企业与求职）并列，风格为科技理性 + 品牌高端。

## 1. 目标与非目标

### 1.1 目标

1. 在同一首页同时承载企业端与求职端转化，首屏双主 CTA 同权。
2. 建立“先可信、后转化”的信息节奏：先价值主张，再信任数据，再双侧路径。
3. 统一视觉语气为科技理性 + 品牌高端，减少当前信息密度不均与视觉漂移。
4. 保持移动端可读性与可点击性，确保双 CTA 在窄屏下同样高效。

### 1.2 非目标

1. 不改后端接口协议与数据库结构。
2. 不引入新的 UI 依赖库或外链图标资源。
3. 不在本次重构中扩展企业/求职业务流程功能，仅优化落地页信息架构与转化路径。

## 2. 设计原则

1. 双目标并列，不做“隐性主次”：企业与求职入口同层级展示。
2. 可证据化表达：文案尽量使用结果性与可量化叙述。
3. 强层级、弱噪音：保留必要动效，不使用炫技型视觉干扰主流程。
4. 一致性优先：按钮、卡片、间距、图标语法全页统一。

## 3. 信息架构（7 个区块）

1. Hero 首屏（双主 CTA 并列）
2. 信任背书条（核心指标）
3. 企业价值流（三步）
4. 求职价值流（三步）
5. 双侧能力矩阵（企业能力 vs 求职能力）
6. 案例/口碑区（企业侧 + 求职侧）
7. 最终双 CTA 收口

## 4. 区块设计明细

### 4.1 Hero（双主 CTA）

- 主标题：AI 图谱驱动的人岗精准连接（结果导向）。
- 副标题：强调“企业更快招到合适的人，人才更快找到匹配的岗”。
- 双主按钮（同权）：
  - 免费发布职位（企业）
  - 立即找工作（求职）
- 按钮下方微文案：
  - 面向招聘方
  - 面向求职者
- 右侧视觉：保留高端数据卡，指标混合企业价值与人才价值。

### 4.2 信任背书条

- 放置在首屏后第一屏区域，优先建立可信度。
- 指标建议：活跃企业数、活跃职位数、平均匹配时长、投递响应率。
- 视觉：高对比数字 + 低噪声标签，避免过度装饰。

### 4.3 企业价值流（三步）

- Step 1 发布职位
- Step 2 AI 推荐候选人
- Step 3 面试协同与转化
- 每步必须有“结果收益文案”而非流程描述。

### 4.4 求职价值流（三步）

- Step 1 上传简历
- Step 2 图谱诊断
- Step 3 精准岗位推荐
- 每步同样输出结果型收益点（效率、匹配度、反馈速度）。

### 4.5 双侧能力矩阵

- 左列：企业侧能力（筛选效率、协同、渠道触达、数据回看）。
- 右列：求职侧能力（岗位匹配、能力补全、投递反馈、成长路径）。
- 要求“对照可读”，避免单边陈列。

### 4.6 案例与口碑

- 企业案例 1 条：招聘效率提升/到岗周期缩短。
- 求职案例 1 条：投递命中率/面试回访率提升。
- 可补充品牌 Logo 或短评作为背书。

### 4.7 最终双 CTA 收口

- 再次并列双主 CTA，与首屏文案逻辑一致。
- 增加低风险承诺微文案（如“3 分钟开始使用”）。

## 5. 交互设计

1. 首屏双 CTA 都是一级按钮，不使用主次按钮语义。
2. 桌面端可加中段轻量悬浮双 CTA 条；移动端改为底部安全区固定入口。
3. 动效只保留三类：首屏渐入、卡片轻位移、数字区淡入。
4. CTA hover 不使用夸张动画，时长控制在 120ms-160ms。

## 6. 视觉与文案系统

### 6.1 视觉基调

- 深蓝背景层 + 亮蓝强调层 + 中性色信息层。
- 使用克制的渐变与玻璃态，不做强反光和高饱和噪点。
- 阴影采用短深阴影，避免“悬浮塑料感”。

### 6.2 排版规范

- Hero 标题控制在 2 行内。
- 正文统一 16-18px 区间。
- 数字指标采用统一格式（如 92.4%、12,480）。

### 6.3 组件语法

- 按钮：统一圆角、边框、focus ring。
- 卡片：统一边框透明度、hover 位移、阴影增量。
- 图标：继续使用当前 Lucide + 本地 Material Symbols 体系。

### 6.4 文案语气

- 少形容词、多结果：强调效率、匹配准确度、响应速度。
- 避免“了解更多”类弱动词，改为明确动作语义。

## 7. Uiverse 借鉴点（转化为本项目可执行规则）

来源：
- https://www.uiverse.io/
- https://uiverse.io/tags/component

借鉴并落地：

1. 双层 CTA 结构
- 首屏双 CTA + 区块内再次给动作入口，缩短转化路径。

2. 标签筛选心智
- 引入角色/场景标签（企业招聘、校园招聘、社招、热门城市等），点击即定位分流内容或带参跳转。

3. 卡片墙一致交互
- 中段卡片统一微交互规则，减少视觉噪音与风格散乱。

4. 高密度信任数字
- 在首屏后立即展示关键数据，优先建立“理性可信”。

5. 新鲜度提示
- 增加“本周新增岗位/企业”轻量信息条，强化平台活跃度认知。

6. 社区/背书模块化
- 用企业 Logo + 双侧短案例替代单向营销文案。

不直接照搬：

1. 过多风格混搭组件。
2. 以炫技为主的重特效 loader。

## 8. 技术改造边界

1. 仅改前端落地页相关实现，不改后端接口。
2. 保持 Next.js + React + Tailwind + framer-motion。
3. 不新增依赖，不新增外链图标。
4. 优先收敛首页单一实现源，避免多实现长期漂移。

## 9. 测试策略（TDD）

### 9.1 RED

先写失败测试，覆盖：
1. 首屏双主 CTA 文案存在且同层级。
2. 企业价值流与求职价值流模块同时存在。
3. 信任背书区至少 4 项指标。
4. 终段再次出现双 CTA。
5. 移动端 CTA 为纵向堆叠结构（结构断言）。

### 9.2 GREEN

以最小实现通过测试：
1. 先完成结构与可访问性，再补视觉细节。
2. 确保 CTA 跳转路径符合既有路由策略。

### 9.3 REFACTOR

1. 抽取复用组件（如 Hero、TrustBar、DualFlow、FinalCta）。
2. 清理重复 class 组合，统一样式语法。

### 9.4 验证

前端改动完成后执行：
1. npm run build
2. npm run test:run

## 10. 风险与应对

1. 风险：双目标导致信息过载。
- 应对：区块顺序固定为“先可信后分流”，并限制每屏信息数量。

2. 风险：现有首页存在多实现漂移。
- 应对：明确单一实现源并同步回归测试。

3. 风险：动效过多影响性能与转化。
- 应对：仅保留三类必要动效，控制时长与触发频率。

## 11. 实施准备结论

该设计满足已确认约束：
1. 双主 CTA 并列。
2. 风格为科技理性 + 品牌高端。
3. 范围为全量重构首页信息架构（不扩业务能力）。

# 登录过期强制退出设计（用户/企业/管理端）

## 背景
当前前端 `apiClient` 仅在 HTTP 401 时执行登出与跳转。后端在部分接口返回 HTTP 200 + 业务包装 `{ code: 401, message: "未登录或登录已过期" }`，导致页面仅显示加载失败，未触发强制退出。

## 目标
当出现未登录/登录过期（HTTP 401 或业务码 401）时，统一执行：
1. 清空当前域登录态
2. 跳转对应登录页（用户/企业 `/login`，管理 `/admin/login`）

## 方案
1. 在 `frontend/src/lib/api/client.ts` 抽取统一 `handleUnauthorized` 方法。
2. 响应成功拦截器中处理包装响应：
   - `code===200` 继续解包
   - `code===401` 执行 `handleUnauthorized` 后 reject
   - 其他非 200 继续 reject
3. 响应失败拦截器保持 HTTP 401 走 `handleUnauthorized`。
4. 新增单测覆盖“包装响应 code=401”触发当前域 logout（用户域 + 管理域）。

## 错误处理
- 对于非 401 的业务失败维持原行为，只返回错误给调用方。
- 对于 401，无论来源（HTTP 状态或包装码）都立即强退，避免页面停留在假登录状态。

## 测试策略
- 先写失败测试：包装码 401 时应调用对应域 `logout`。
- 实现最小修复通过测试。
- 回归构建/测试并执行浏览器验证。

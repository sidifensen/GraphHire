# 管理端夜间模式视觉修正设计

## 背景
当前管理端暗色模式中，菜单、主内容区、卡片层级不符合预期，导致对比度和信息层次不稳定。

## 目标
1. 左侧菜单栏在暗色模式下为黑色。
2. 右侧主内容区在暗色模式下为灰色底。
3. 内容卡片在暗色模式下为黑色底。

## 方案
- 在 `AdminSidebar` 增加明确暗色类：`dark:bg-black`。
- 在 `AdminShell` 的 `main` 增加语义类 `admin-main-surface` 与 `dark:bg-gray-700`。
- 在全局样式中增加局部覆写：仅对 `admin-main-surface` 内的 `bg-white` 设为黑色，避免影响侧栏。

## 风险与回归点
- 风险：全局 `bg-white` 覆写可能影响弹窗/表格等组件。
- 控制：将黑色卡片规则限定在 `admin-main-surface` 下。

## 测试策略
- 新增/更新页面壳测试：验证侧栏与主内容区暗色类名存在。
- 手工浏览器验收：检查 dashboard 页在暗色模式下三块颜色是否符合目标。

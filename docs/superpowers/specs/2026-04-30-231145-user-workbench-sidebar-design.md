# 桌面端“我的”工作台侧栏化改造设计

**日期**: 2026-04-30  
**范围**: 用户端 5 个页面（`/profile`、`/personal-info`、`/resume/manage`、`/applications`、`/skill-graph`）  
**约束**: 保留顶部 `TopNav`；仅调整桌面端信息架构与视觉表达；移动端行为与布局保持现状。

## 1. 背景与目标

当前用户端“我的”相关页面在桌面端大量使用大圆角卡片堆叠，导致：
- 横向空间利用率低，信息密度偏低
- 页面间结构一致性不强（入口和内容层级混杂）
- 桌面端操作路径不够“工作台化”

本次改造目标：
- 形成统一的桌面端工作台信息架构：`左侧菜单栏 + 右侧页面内容`
- 将“我的”入口菜单从首页卡片区抽离为全局侧栏
- 逐页去除重卡片化表达，改为列表/分区/行级操作
- 不改变业务逻辑、接口调用与关键交互语义

## 2. 设计决策

### 2.1 布局决策

- 桌面端断点：`>= 1024px`（`lg`）
- 全局结构（仅 5 个目标页面启用）：
  - 左栏：固定菜单（与“我的”入口一致）
  - 右栏：当前路由主内容
- 顶部 `TopNav` 保留，作为页面标题与回退行为容器

### 2.2 菜单决策

左侧菜单固定为：
- 个人资料 → `/personal-info`
- 简历管理 → `/resume/manage`
- 投递记录 → `/applications`
- 我的图谱 → `/skill-graph`
- 账号设置 → `#`（占位，保持现状）

激活规则：按当前 pathname 精确或前缀匹配高亮。

### 2.3 视觉决策

桌面端从“卡片中心”改为“工作区中心”：
- 减少超大圆角和浮起阴影
- 强化分割线、分区标题、行内操作
- 列表页优先使用“行/列信息块”而非多列卡片宫格
- 移动端样式类保留，不做回归风险扩散

## 3. 分页面改造方案

### 3.1 `/profile`（我的首页）

- 移除当前右侧菜单宫格（菜单迁移到左栏）
- 保留用户身份信息、统计、求职意向，但重排为右侧“概览工作区”
- 夜间模式开关与退出登录作为行级工具区，避免孤立大卡片
- 维持现有数据来源（auth store + `personApi.getProfile`）

### 3.2 `/personal-info`

- 将两块大卡片表单改为右侧连续分区表单（基础信息/求职意向）
- 头像上传、保存按钮逻辑不变
- 保持字段、校验、提交 payload 不变

### 3.3 `/resume/manage`

- 将简历卡片网格改为桌面端行式列表
- 每行呈现：文件名、状态、创建时间、默认标签、操作区
- 保持解析、设默认、删除、预览、上传逻辑不变

### 3.4 `/applications`

- 状态 tabs 保留
- 投递记录由卡片网格改为行式记录
- 每行展示：职位、公司、匹配度、状态、投递时间、撤回/查看操作
- 保持撤回逻辑与筛选逻辑不变

### 3.5 `/skill-graph`

- 保留图谱画布主体
- 右侧统计块改为更扁平的“侧边信息面板”
- 保持图节点、动画、按钮行为不变

## 4. 组件与文件边界

新增与调整重点：
- 新增用户工作台侧栏组件（仅桌面端显示）
- 在 5 个页面中引入统一桌面壳层（侧栏 + 内容）
- 页面内部主要调整容器 className 与结构层级，不触碰 API 合约

预期影响文件：
- `src/app/(user)/profile/page.tsx`
- `src/app/(user)/personal-info/page.tsx`
- `src/app/(user)/resume/_components/ResumeManageContent.tsx`
- `src/app/(user)/applications/page.tsx`
- `src/app/(user)/skill-graph/page.tsx`
- （新增）`src/app/(user)/_components/UserWorkbenchSidebar.tsx`（命名以实现阶段为准）

## 5. 风险与兼容性

- 主要风险：视觉结构重排引发已有页面测试选择器失效
- 控制策略：
  - 优先保持按钮文案与 aria-label 不变
  - 必要时同步更新页面测试快照/查询点
  - 不改 API 调用与状态管理路径

## 6. 测试与验收策略

- 单元/页面测试：覆盖 5 页关键可见元素与关键交互
- 构建与测试：`npm run build`、`npm run test:run`
- 最终联调：按仓库要求补充后端 `mvn compile`、`mvn test`
- 浏览器验证：通过 CDP 打开对应页面，检查桌面端左栏与右侧内容联动

## 7. 非目标

- 不引入新的业务能力（例如真实账号设置页）
- 不改移动端底部导航与移动端页面结构
- 不修改后端接口或数据模型

# 邮箱验证码发送体验修复设计

## 目标
修复注册页“发送验证码”在后端慢返回时前端误报超时，以及英文错误提示问题，统一为中文可理解提示。

## 根因
1. 后端 `AuthAppService.sendVerifyCode` 同步发送邮件，SMTP 慢连接导致接口耗时超过 30 秒。
2. 前端 `apiClient` 默认超时 30000ms，超时后直接显示 axios 英文消息。
3. 前端注册页对错误信息缺少 timeout 场景中文归一化。

## 方案
1. 后端：对邮件发送异常进行捕获，返回统一业务异常中文文案 `验证码发送失败，请检查邮箱地址或稍后重试`，并清理已写入的验证码缓存。
2. 后端：配置 SMTP 连接/读/写超时（8 秒），避免单次请求长时间阻塞。
3. 前端：在 `apiClient` 中将 `ECONNABORTED` 和 timeout 文案映射为中文 `请求超时，请稍后重试`。
4. 前端：`sendVerifyCode` 请求单独放宽超时到 60000ms，降低误报概率。
5. 测试：新增后端发送失败测试、前端 timeout 映射测试和注册页提示测试。

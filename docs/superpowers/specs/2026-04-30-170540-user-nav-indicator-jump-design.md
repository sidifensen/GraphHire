# 用户端顶部导航激活态漂移修复设计

## 背景
- 桌面端用户首页滚动后切换顶部菜单（首页/职位/公司/我的）时，激活态背景有概率从页面下方漂移到目标菜单，而不是在菜单之间水平平滑移动。

## 根因
- 现有实现依赖 `framer-motion` 的 `layoutId` 在不同 `NavLink` 之间做共享布局动画。
- 在页面切换与滚动位置变化叠加时，共享布局测量会出现错误起点，导致激活态沿 Y 轴跳变。

## 方案
- 移除导航项内部的 `layoutId` 共享动画。
- 改为“导航容器内单实例指示器”：
  - 通过激活菜单项与容器的 `getBoundingClientRect` 计算 `x` 与 `width`。
  - 使用 `motion.div` 仅动画 `x`、`width`，固定 `y=0`，强制水平过渡。
  - 监听路由变化与 `resize`，实时更新指示器。

## 影响范围
- `frontend/src/app/(user)/_mock/components/Navbar.tsx`
- `frontend/src/lib/ui/nav-indicator.ts`
- 前端测试新增导航指示器计算回归用例。

## 风险与回退
- 风险：首次渲染前指示器宽度可能为 0。
- 缓解：在计算函数中对异常宽度回退为 `1`，避免指示器丢失。
- 回退策略：若新方案异常，可回退到旧 `layoutId` 实现。

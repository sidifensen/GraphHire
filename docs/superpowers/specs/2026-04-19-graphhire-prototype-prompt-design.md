# GraphHire Prototype Prompt Optimization Design

**Date:** 2026-04-19
**Status:** Approved
**Scope:** Optimize the existing GraphHire prototype-generation prompt into one single high-utility prompt for AI prototype websites, and save the result into a new folder with an explicit GPT-5.4 attribution.

---

## 1. Goal

Produce one consolidated prompt that can be pasted directly into an AI prototype-generation website so it can generate a complete, coherent, multi-page GraphHire prototype in one run.

The new prompt must improve over the current draft in three ways:

1. It must be easier for a generation model to follow than the current long outline-plus-wireframe format.
2. It must balance business completeness and visual direction rather than leaning too hard toward either one.
3. It must explicitly constrain the generator so the output is a full prototype system instead of a handful of disconnected screens.

---

## 2. Deliverables

The work produces the following deliverables:

1. A new prompt file under `docs/原型图/gpt5.4/`.
2. A short design spec in `docs/superpowers/specs/`.
3. A matching acceptance criteria document in `docs/superpowers/acceptance/`.
4. A lightweight implementation plan in `docs/superpowers/plans/`.

The original prompt files under `docs/原型图/` remain unchanged.

---

## 3. Prompt Design Strategy

The prompt will use a "product director" structure rather than a raw feature inventory. This means the generated document tells the prototype website:

1. What it is generating.
2. What visual and interaction language it must keep consistent.
3. Which pages must exist for each user role.
4. Which business states and transitions must appear.
5. Which output mistakes are forbidden.

This structure is more reliable for one-shot prototype generation than ASCII wireframes because prototype sites typically respond better to hierarchical instructions, explicit constraints, and page-level priorities than to text diagrams.

---

## 4. Content Structure of the New Prompt

The prompt will be organized into these sections:

### 4.1 Generation Mission

Defines GraphHire as an AI recruitment platform and clearly states that the output must be a complete multi-page prototype for three terminals:

1. User side
2. Enterprise side
3. Admin side

It will explicitly require one coherent design system with role-specific variations.

### 4.2 Global Visual Direction

Defines the expected visual personality:

1. Modern, professional, trustworthy
2. Data-product feel with AI-assisted intelligence cues
3. Clean cards, strong information hierarchy, sufficient whitespace
4. Balanced visual polish and business practicality

This section replaces scattered style notes with one reusable design language.

### 4.3 Shared Interaction and State Rules

Defines cross-page expectations:

1. Loading, empty, success, error states
2. Hover, button, and page transition behavior
3. Responsive behavior for desktop, tablet, and mobile
4. Navigation consistency and page jump logic

This prevents the generator from making static-looking pages with no operational states.

### 4.4 Page Generation Requirements

Groups pages by terminal and describes each page using:

1. Page objective
2. Required modules
3. Key interaction points
4. Important states or variants

This keeps the output complete without falling back into low-signal wireframe blocks.

### 4.5 Hard Constraints

Defines non-negotiable output constraints:

1. Must generate all required pages, not selected samples
2. Must preserve visual consistency across all screens
3. Must show important business status flows
4. Must avoid placeholder-only outputs and generic SaaS templates
5. Must produce a prototype suitable for product review and front-end reference

## 6. Output File Rules

The final prompt file must:

1. Live in a new folder: `docs/原型图/gpt5.4/`
2. Include a file header stating it was generated by GPT-5.4
3. Be written in Chinese because the current prompt assets and project documentation are in Chinese
4. Be directly usable as a paste-ready prompt for an AI prototype-generation website

---

## 7. Non-Goals

This task does not:

1. Replace existing prompt files
2. Generate actual HTML or coded pages
3. Change business scope beyond what is already established
4. Introduce new product modules unrelated to the current GraphHire prototype set

---

## 8. Validation

The work is correct if:

1. A new folder exists under `docs/原型图/`.
2. A new GPT-5.4-labeled prompt file exists in that folder.
3. The prompt reads as one complete generation brief rather than fragmented notes.
4. The prompt clearly covers all three terminals and their major pages.
5. The original prompt files remain untouched.

# 管理端登录页开发模式自动填充设计

**日期：** 2026-04-27  
**范围：** `frontend/src/app/admin/login/page.tsx` 与其页面测试

## 背景

管理端登录页当前每次本地开发都需要手工输入测试账号，影响联调效率。

## 目标

在 `development` 环境下，管理端登录页默认填充测试账号与密码；非开发环境保持空值，避免生产环境暴露默认凭据。

## 方案

1. 在管理端登录页声明 `isDev = process.env.NODE_ENV === 'development'`。
2. 新增开发测试凭据常量：
   - `username: admin`
   - `password: 123456`
3. 将 `username` 与 `password` 的初始 `useState` 值切换为“开发环境填充、其他环境为空”。
4. 通过页面测试覆盖开发环境预填充行为。

## 数据流与影响

- 仅影响页面初始渲染值；
- 不改变登录请求参数结构；
- 不影响提交、鉴权写入与跳转逻辑。

## 风险与规避

- 风险：误在非开发环境出现默认值。  
  规避：通过 `NODE_ENV` 严格判断，仅 `development` 生效，并在测试中断言。

## 测试策略

1. 新增失败测试：`NODE_ENV=development` 时账号/密码默认值应为测试凭据。
2. 保持既有成功登录与失败提示测试通过。

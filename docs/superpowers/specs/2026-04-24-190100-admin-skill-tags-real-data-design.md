# 管理端标签管理对接真实技能表设计

## 背景
当前标签管理页仍展示“分类/状态/引用次数”等后端技能表不存在的字段，筛选区也包含无效筛选项，导致页面和真实接口契约不一致。

## 目标
1. 标签列表仅依赖后端真实字段：`id`、`name`、`synonyms`。
2. 列表展示同义词实际内容，不展示分类与引用次数。
3. 列表右侧提供可见的“编辑”“删除”按钮。
4. 顶部筛选区仅保留一个搜索框，移除分类/状态筛选和筛选按钮。

## 数据流
1. 页面通过 `adminApi.getSkillList({ keyword, page, pageSize })` 拉取数据。
2. 前端将 `synonyms` 做空值兜底为 `[]`。
3. 编辑操作调用 `adminApi.updateSkillTag(id, { name })`。
4. 删除操作调用 `adminApi.deleteSkillTag(id)`。

## 容错与交互
1. `synonyms` 为 `null` 时显示“无”。
2. 搜索框输入时重置到第一页并触发重新加载。
3. 删除前增加确认提示，避免误操作。

## 测试策略
1. 页面测试验证：只保留搜索框、无筛选按钮、无分类/引用次数列。
2. 页面测试验证：同义词内容可见。
3. 页面测试验证：编辑与删除按钮可触发对应 API。
4. 页面测试验证：`synonyms = null` 时回退显示“无”。

# 企业端与用户端头像下拉退出设计

**日期**: 2026-04-30
**范围**: 前端（企业端/用户端 Mock Shell 顶部导航）

## 背景
当前企业端与用户端的桌面端头像区域展示了用户信息，但缺少统一的头像下拉退出入口；企业端移动端顶部头像也不是可点击退出入口，导致用户无法按“点击头像 -> 退出登录”的方式完成登出。

## 目标
1. 企业端桌面端：点击右上角头像区域显示下拉菜单，提供“退出登录”。
2. 企业端移动端：点击顶部头像显示下拉菜单，提供“退出登录”。
3. 用户端桌面端：点击右上角头像区域显示下拉菜单，提供“退出登录”。
4. 用户端移动端：保留现有可用退出路径，并确保新增/现有退出入口都调用统一登出逻辑并生效。

## 方案
1. 在 `frontend/src/app/enterprise/_mock/components/TopNav.tsx` 增加头像菜单状态、点击外部关闭逻辑、桌面/移动双位置菜单触发器，并复用 `logoutWithServerInvalidation` 执行退出。
2. 在 `frontend/src/app/(user)/_mock/components/Navbar.tsx` 增加头像菜单状态、点击外部关闭逻辑，在桌面端头像按钮中加入下拉菜单并复用统一退出逻辑。
3. 不新增后端接口，继续复用 `authApi.logout` + 本地 store 清理流程。

## 交互与错误处理
1. 退出按钮点击后，无论服务端退出是否异常，前端都应清理本地登录态并跳转（现有 `logoutWithServerInvalidation` 已满足）。
2. 点击菜单外区域自动关闭菜单，避免悬浮菜单残留。
3. 头像加载失败回退默认图标逻辑保持不变。

## 测试策略
1. 先补单测（RED）：验证点击头像后出现“退出登录”；点击“退出登录”会调用统一登出方法。
2. 实现最小代码使测试通过（GREEN）。
3. 回归执行前端构建与测试、后端编译与测试。
4. 用 CDP 浏览器对企业端/用户端的 PC + 手机视口做头像退出交互验证。

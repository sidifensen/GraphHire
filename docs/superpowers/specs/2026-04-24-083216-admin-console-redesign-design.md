# 管理端全页面重构设计方案（基于 `docs/graphhire-图谱智聘管理端`）

## 1. 目标与范围

### 1.1 重构目标
- 以原型目录 `docs/graphhire-图谱智聘管理端` 为视觉与结构基准，重写管理端 6 个页面：
  - 登录页：`/admin/login`
  - 仪表盘：`/admin/dashboard`
  - 企业审核：`/admin/enterprise-review`
  - 用户管理：`/admin/users`
  - 标签治理：`/admin/skill-tags`
  - 任务监控：`/admin/task-monitor`
- 页面风格从“Material 控制台”切换为“轻玻璃 + 数据卡片 + 高密度表格”风格。
- 保留现有真实业务行为（接口调用、筛选、分页、批量操作、登录鉴权）。

### 1.2 非目标
- 不修改后端接口协议。
- 不新增管理端业务路由。
- 不调整 `AdminAuthGuard` 的鉴权判定逻辑。

---

## 2. 现状与差距

### 2.1 现状
- 管理端页面位于 `frontend/src/app/admin/**/page.tsx`。
- 已具备完整 Admin API：
  - `getDashboardStats`
  - `getCompanyAuthList` / 审核通过拒绝 / 批量审核
  - `getUserList` / 用户状态变更 / 批量禁用 / 用户详情
  - `getSkillList`
  - `getTaskList` / 重试
  - `login`

### 2.2 与原型差距
- 页面视觉层次、信息卡片密度、表格样式和顶部操作区与原型不一致。
- 现有页面整体结构可用，但视觉系统不统一。

---

## 3. 重构设计

## 3.1 结构设计
- 新增统一管理端壳组件：`AdminShell`
  - 组合侧边栏、顶部栏、滚动主区域。
  - 每页只关注业务内容，减少重复布局代码。
- 新增通用组件：
  - `AdminTopbar`：搜索框、主题按钮（仅占位展示）、管理员头像区
  - `AdminDataTable`：统一头部、行 hover、分页区
  - `AdminStatCard`：指标卡样式统一
- 保留现有 `AdminSidebar` 的导航信息与路由，不改菜单项。

## 3.2 页面级设计

### 3.2.1 登录页 `/admin/login`
- 改为左右分栏（品牌文案 + 登录卡片）布局。
- 保留行为：
  - 调用 `adminApi.login`
  - 使用 `adminAuthStore.setAuth`
  - 登录成功跳转 `/admin/dashboard`
  - 失败展示错误文案

### 3.2.2 仪表盘 `/admin/dashboard`
- 顶部公告 + KPI 卡 + 趋势图 + 待办列表 + 热门技能/系统动态区。
- 数据来源保持 `getDashboardStats`。
- 保留加载/失败态。

### 3.2.3 企业审核 `/admin/enterprise-review`
- 增加“统计卡 + 筛选条 + 表格”三段结构。
- 保留能力：
  - 状态筛选
  - 关键词检索
  - 单条通过/拒绝
  - 批量通过/拒绝
  - 分页切换

### 3.2.4 用户管理 `/admin/users`
- 增加“筛选控制栏 + 数据表 + 详情弹窗”结构。
- 保留能力：
  - 类型/状态过滤、关键词检索
  - 批量禁用
  - 单用户状态切换（启用/禁用/解锁）
  - 详情弹窗请求 `getUserDetail`

### 3.2.5 标签治理 `/admin/skill-tags`
- 增加“总览指标 + 筛选卡 + 标签表格 + 侧栏建议”布局。
- 保留能力：
  - 关键词与分类筛选
  - 分页/加载更多逻辑

### 3.2.6 任务监控 `/admin/task-monitor`
- 增加“状态卡 + 任务队列 + 错误日志行”布局。
- 保留能力：
  - 刷新
  - 单任务重试
  - 批量重试
  - 失败任务错误信息展示

---

## 4. 数据与交互约束

- 不更改 `adminApi` 方法签名与返回类型。
- 所有页面必须保留：
  - `loading` 过程可见
  - 请求失败可重试
  - 空数据态可识别
- 含破坏性操作（拒绝、批量拒绝）继续通过确认输入（`prompt`）收集理由，避免行为倒退。

---

## 5. 测试设计（TDD）

- 先更新页面测试，确保以下场景在当前实现下失败，再进行生产代码重写：
  - 登录页：新文案与按钮、错误信息可见
  - 仪表盘：新标题与关键模块渲染
  - 企业审核：筛选区与数据区渲染
  - 用户管理：详情弹窗入口可见
  - 标签治理：核心标题与列表渲染
  - 任务监控：状态卡与任务项渲染
- 重构完成后运行全量前端单测。

---

## 6. 风险与应对

- 风险：UI 重构影响已有测试快照/文本断言。
  - 应对：先改测试断言为语义关键点，不依赖脆弱类名。
- 风险：重构过程中误删 API 行为。
  - 应对：每页保留原调用路径，增加交互测试覆盖关键按钮。
- 风险：页面改动大导致样式不一致。
  - 应对：提炼公共组件，减少重复 CSS 片段。

---

## 7. 交付结果

- 管理端 6 页 UI 与结构按原型风格完成重写。
- 业务交互能力与鉴权逻辑保持可用。
- 测试、构建与浏览器验收通过后提交。

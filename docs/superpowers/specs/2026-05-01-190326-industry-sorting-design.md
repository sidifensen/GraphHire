# 行业管理排序能力设计

**日期：** 2026-05-01  
**范围：** 管理端行业管理页面（后端 + 前端）

## 背景

当前行业排序字段 `sortOrder` 在历史数据中常见值为 `0,10,20...`，页面直接展示后给用户造成“十位跳号”的感知。当前模块缺少以下能力：

1. 列表按不同字段升降序查看
2. 管理员直接调整行业相对顺序并持久化
3. 调整后排序值连续化（0 到 N-1）

## 目标

1. 支持行业列表按 `name/sortOrder/updatedAt` 的升序或降序查询
2. 支持行业上移/下移，持久化后顺序立即生效
3. 每次移动后将全量行业 `sortOrder` 归一化为连续整数（0..N-1）
4. 前端支持表头排序交互与上移/下移动作

## 非目标

1. 不实现拖拽排序
2. 不修改企业侧行业选项接口结构（仅受排序结果影响）
3. 不做批量排序导入

## 方案

### 1. 后端接口扩展

- `GET /admin/industry/list`
  - 新增参数：
    - `sortBy`: `name | sortOrder | updatedAt`
    - `sortDir`: `asc | desc`
  - 默认：`sortBy=sortOrder`，`sortDir=asc`

- 新增 `PUT /admin/industry/{id}/move`
  - 请求体：`{ "direction": "UP" | "DOWN" }`
  - 行为：
    1. 在“当前默认顺序（sortOrder asc, id asc）”中定位目标行业
    2. 若可移动，与相邻元素交换逻辑位置
    3. 对完整列表重写 `sortOrder=0..N-1`
    4. 返回移动后的行业项

### 2. 领域与仓储调整

- `IndustryRepository` 支持带排序参数的查询方法
- `IndustryAppService` 支持按动态排序列查询
- 新增移动方法，内部执行重排连续排序

### 3. 前端交互

- 行业表格列头：
  - `行业名称/排序/更新时间` 可点击切换 `asc/desc`
  - 排序状态显示（文本箭头）
- 行操作新增：
  - `上移`、`下移` 按钮
  - 触发 move 接口后刷新
- 新增行业时默认排序值：当前页数据最大 `sortOrder + 1`

## 错误处理

1. `id` 不存在：返回业务异常“行业不存在”
2. 到达边界仍移动（首行上移/末行下移）：无变更，返回当前项
3. 非法排序字段或方向：回退默认排序

## 测试策略

### 后端

1. `AdminControllerTest`：
   - 行业列表接口透传排序参数
   - move 接口调用应用服务
2. `AdminAppServiceTest`：
   - 行业列表按排序参数返回
3. `IndustryAppServiceTest`（新增）：
   - move up/down 触发连续重排
   - 边界移动保持不变

### 前端

1. 行业页面测试（新增）：
   - 初始加载参数正确
   - 点击列头切换排序并重新请求
   - 点击上移/下移调用对应 API

## 兼容性

- 不破坏现有 `industry/list` 参数兼容性
- 旧前端不传排序参数时行为与当前一致（默认按 sortOrder asc）

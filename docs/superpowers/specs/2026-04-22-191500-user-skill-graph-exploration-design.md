# 用户端能力图谱探索交互升级设计

**日期：** 2026-04-22  
**状态：** 草案已确认（待实现）  
**范围：** `frontend/src/app/(user)/skill-graph/page.tsx`

---

## 1. 目标与范围

### 1.1 目标

在不新增后端业务字段、不改变业务流程的前提下，升级用户端能力图谱页面的视觉表现与探索交互，重点实现：

1. 图谱可平移与缩放
2. 技能节点可拖拽
3. 图谱整体更具“探索感”和沉浸式视觉效果

### 1.2 非目标

1. 不新增行动建议逻辑
2. 不新增后端接口或字段
3. 不改变简历上传、匹配、通知等业务链路

---

## 2. 交互设计

### 2.1 画布交互

1. 鼠标拖动画布平移
2. 滚轮缩放，限制最小/最大缩放倍率
3. 双击节点触发聚焦：节点居中并适度放大视图
4. 提供“重置视图”按钮恢复默认视角

### 2.2 节点交互

1. 技能节点支持拖拽重排
2. 松开后保留本次会话内位置
3. 悬浮节点时高亮该节点与关联边
4. 非关联节点降透明度，减少视觉噪声

### 2.3 动效策略

1. 首次加载时节点由中心向外展开
2. 拖拽时连线实时跟随
3. 聚焦时相关节点与边增强，其余淡出
4. 动效采用轻阻尼，避免晕眩和过度跳动

---

## 3. 视觉设计

### 3.1 风格方向

1. 图谱主区采用沉浸式深色画布
2. 技能节点采用霓虹描边与轻呼吸动效
3. 连线采用细线渐变表现关系网络
4. 保持与全站视觉语言一致，避免割裂

### 3.2 布局结构

1. 保持现有左右布局
2. 左侧为图谱主舞台
3. 右侧由“建议卡”转为“图谱状态卡”，展示节点总数、连接数、当前聚焦节点信息

### 3.3 节点分层

1. 中心用户节点最大，作为视觉锚点
2. 技能节点按权重分三档大小
3. 悬浮显示技能名与关联度浮层

---

## 4. 技术方案

### 4.1 渲染引擎

1. 使用 `react-force-graph-2d` 替换当前手工绝对定位图谱绘制
2. 复用现有 React 页面状态流与加载流程

### 4.2 数据映射

基于当前 `personApi.getGraph()` 的 `skills` 列表在前端组装图数据：

1. `user` 中心节点 1 个
2. `skill` 节点 N 个
3. 中心到技能的基础连线 N 条
4. 可选技能间弱连接（仅视觉连贯用途，前端规则生成，不持久化）

### 4.3 页面状态

保留原有：

1. `loading`
2. `error`
3. `empty`

新增交互状态：

1. `selectedNode`
2. `hoverNode`
3. `isDragging`

### 4.4 边界与约束

1. 节点位置仅在当前会话内有效，不落库
2. 不依赖新后端字段
3. 保持现有空态、异常态文案可读性

---

## 5. 风险与应对

### 5.1 主要风险

1. 力导向布局初始抖动影响观感
2. 节点数量增加后画布性能下降
3. 深色沉浸画布与站内配色冲突

### 5.2 控制策略

1. 调整仿真参数（速度衰减、碰撞半径、中心引力）降低抖动
2. 控制单屏节点密度并启用层级高亮
3. 复用设计 token，限制自定义色域范围

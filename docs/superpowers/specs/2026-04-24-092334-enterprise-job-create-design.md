# 企业端新增岗位直发设计

**日期**: 2026-04-24  
**范围**: 前端企业端岗位管理

## 背景

当前企业端“职位管理”页中的“发布新职位”按钮链接到 `/enterprise/jobs` 本页，点击后无页面变化。用户期望有“新增岗位”页面，并且提交后直接发布，不走草稿流。

## 目标

1. 企业端提供独立新增岗位页面：`/enterprise/jobs/new`。  
2. 新增岗位表单提交后，执行“创建职位 + 立即发布”。  
3. 成功后返回职位列表并反馈成功状态。  
4. 失败时反馈具体错误，不出现静默失败。

## 方案

### 路由与入口

- 新增页面文件：`frontend/src/app/enterprise/jobs/new/page.tsx`
- 职位管理页按钮从 `/enterprise/jobs` 改为 `/enterprise/jobs/new`

### 提交流程

1. 调用 `POST /company/job` 创建职位，获取 `jobId`
2. 调用 `POST /company/job/{id}/publish` 立即发布
3. 成功后跳转 `/enterprise/jobs?created=1`

### 字段与校验（MVP）

- 必填：职位名称、职位描述、城市、最低薪资、最高薪资
- 选填：部门、招聘人数
- 校验：
  - 必填不能为空
  - 薪资必须为正数
  - 最低薪资不能大于最高薪资

### 错误处理

- 创建失败：展示接口错误
- 发布失败：展示接口错误并允许重试提交
- 校验失败：提交前阻断并展示提示

## 测试策略

1. 先写失败测试（TDD Red）：
   - 职位管理页按钮链接应为 `/enterprise/jobs/new`
   - 新增岗位页提交成功应依次调用 `createJob` 和 `publishJob`
2. 实现最小代码（TDD Green）
3. 补充异常测试并确保全部通过（Refactor）

# 聊天面试通知弹窗组件化（shadcn）设计规格

## 背景与目标

当前企业端聊天页的“面试通知”采用输入区上方内联三输入框，交互拥挤且难以复用。目标是将其改造为可复用组件：点击“面试通知”按钮后打开 shadcn 风格弹窗，在弹窗中填写并发送。

## 范围

- 企业端聊天页：`/enterprise/chat`、`/enterprise/chat/[conversationId]`
- 前端组件重构与交互升级
- 保持现有接口 `chatApi.sendInterviewInvite` 契约不变

## 非目标

- 不改后端接口与字段定义
- 不调整用户端（求职者）输入区交互
- 不引入复杂第三方日期库

## 需求拆解

### 1. 组件化

新增 `InterviewInviteDialog` 独立组件，职责包括：
- 打开/关闭弹窗
- 管理表单状态（时间、地点、备注）
- 时间选择弹层（日期 + 时分）
- 表单校验与提交触发

`ChatWorkspace` 仅负责：
- 传入回调 `onSubmit`
- 传入 loading 状态
- 处理接口调用成功后的消息刷新和全局错误提示

### 2. 弹窗交互

- 点击“面试通知”按钮打开 `Dialog`
- 弹窗字段：
  - 面试时间（必填）
  - 面试地点（必填）
  - 面试备注（选填）
- 点击“面试时间”字段打开 `Popover`：
  - `Calendar` 选择日期
  - `time` 输入选择时分
- 表单提交时将时间格式化为 `yyyy-MM-ddTHH:mm:ss`
- 提交成功后关闭弹窗并清空表单
- 提交失败时保留表单并显示错误

### 3. shadcn 组件策略

采用 shadcn 组合范式并补齐基础组件：
- `Dialog`
- `Popover`
- `Button`
- `Input`
- `Label`
- `Calendar`（轻量组合实现）

新增组件均放置在 `frontend/src/components/ui/`，确保后续页面可复用。

### 4. 数据与校验

- 必填校验：时间与地点非空
- 时间值统一输出 ISO 本地字符串（秒固定 `:00`）
- 接口参数维持：
  - `conversationId`
  - `interviewTime`
  - `location`
  - `remark`

### 5. 测试策略（TDD）

在 `chat-workspace-redesign.test.tsx` 先补失败测试（RED）：
- 企业端点击“面试通知”出现弹窗
- 弹窗内存在字段与确认按钮
- 原内联“确认发送面试通知”不再在输入区常驻
- 点击提交后调用 `sendInterviewInvite`，参数含格式化时间

随后实现（GREEN）并确保测试通过。

## 风险与缓解

- 风险：项目当前未完整引入 shadcn 全套依赖。
- 缓解：采用最小依赖的轻量 ui 封装，避免引入不必要包。

- 风险：时区与时间格式不一致。
- 缓解：前端统一用本地时间拼接字符串并补秒，保持与当前后端兼容。

## 验收标准摘要

- 企业端聊天页“面试通知”改为弹窗填写
- 时间字段点击可打开日期+时间选择面板
- 提交后仍调用原接口并刷新消息
- 前端构建与测试通过

# 职位类型管理新增展示模式设计

## 背景与目标
当前管理端仅支持“树+详情”和“树形表格”。本次新增两种更适合深层维护和批量定位的视图：
1. 路径面包屑视图：左侧仅显示当前节点同级，顶部显示完整祖先路径。
2. 分栏级联视图：三列分别展示一级、二级、三级，点击左列联动刷新右列。

## 方案结论（采用）
在现有 `viewMode` 基础上扩展为四种模式，不改变后端接口与数据结构，完全复用已有树数据。
- 新增 `breadcrumb` 模式：通过“当前节点 + 父链路 + 同级节点集合”派生视图。
- 新增 `cascade` 模式：通过“一级选中、二级选中、当前选中”驱动三级列渲染。

## 关键设计
### 1. 路径面包屑视图
- 顶部显示 `祖先1 / 祖先2 / 当前节点`。
- 左侧列表仅显示当前节点同级节点（根节点时显示一级列表）。
- 点击同级节点后刷新面包屑与详情区。

### 2. 分栏级联视图
- 第一列：全部一级节点。
- 第二列：当前一级节点下的二级节点。
- 第三列：当前二级节点下的三级节点。
- 点击任意列节点，自动修正当前选中节点并保持右侧详情同步。

### 3. 状态与兼容性
- 保留原有 `selectedId` 作为单一真实选中源。
- 新增少量派生函数（父链查找、同级查找、分栏数据派生），不引入新依赖。
- 表格模式与原树模式行为保持不变。

## 测试策略
- 页面测试新增用例：
  - 可切换到“路径面包屑视图”，展示 `技术 / 后端开发 / Java`，且左侧不出现非同级根节点。
  - 可切换到“分栏级联视图”，三列标题存在，点击节点后详情区同步。
- 保留原有筛选与操作测试，确保无回归。

## 风险与控制
- 风险：节点切换时多模式派生状态可能错位。
- 控制：全部从 `selectedId` 派生，不额外维护重复真值状态；通过测试覆盖点击链路。

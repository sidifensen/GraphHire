# 管理端登录页渐变修复 — 设计规格

**日期：** 2026-04-21  
**状态：** 已批准（用户要求直接实施）  
**范围：** 仅调整 `frontend/src/app/admin/login/page.tsx` 的视觉层次与配套测试

---

## 1. 问题定义

当前管理端登录页虽然保留了浅色背景，但左上角深蓝径向渐变不明显，且缺少原型卡片右上角的环境光晕，导致页面视觉层次明显弱于原型 `docs/前端原型图/管理端/1.管理端登录页.html`。

## 2. 目标

1. 页面左上角恢复可感知的深蓝色径向渐变。
2. 页面右下角保留浅蓝平衡渐变，维持原型的双角呼应。
3. 登录卡片补回右上角淡蓝环境光晕。
4. 不修改现有登录交互逻辑、接口调用、状态写入和跳转行为。

## 3. 实施方案

### 3.1 页面背景
- 将当前固定背景层替换为更接近原型的多层 `radial-gradient`。
- 左上角渐变使用更深的蓝色与更高覆盖范围，确保肉眼可见。
- 保留浅底色 `#f8f9ff` 作为底板。

### 3.2 卡片氛围
- 在登录卡片内部新增右上角模糊光晕装饰层。
- 光晕不遮挡表单，层级位于内容下方。

### 3.3 可测试性
- 为管理端登录页新增页面测试。
- 测试验证：
  - 页面存在品牌区与登录标题。
  - 背景层含左上角深蓝径向渐变标识。
  - 卡片内部存在环境光晕装饰节点。

## 4. 非目标

- 不重构管理端布局。
- 不新增验证码、忘记密码流程或 remember-me 逻辑。
- 不修改后端接口。

## 5. 验收口径

- 页面渲染后可见更明显的左上角深蓝渐变。
- 登录卡片右上角存在淡蓝色环境光晕。
- 相关 Vitest 用例通过。
- 前端 `next build` 通过。

# 用户端职位详情匹配度圆环展示设计

- 日期：2026-04-25
- 范围：`frontend/src/app/(user)/jobs/[id]/page.tsx`

## 背景
当前“综合匹配度”使用纯文本展示，信息可读性一般。需要改为更直观的圆环进度展示，并保留原有匹配明细与投递流程。

## 目标
在匹配成功状态下，将“综合匹配度 XX%”替换为环形进度圈，中心显示百分比数字，右侧显示“综合匹配度”标签。

## 方案
1. 在当前页面文件内新增 `MatchScoreRing` 组件（局部复用，避免过度抽象）。
2. 使用 SVG 两层圆：底环 + 进度环；通过 `strokeDasharray`/`strokeDashoffset` 控制进度。
3. 分数做边界保护（0-100），避免异常值造成 UI 失真。
4. 保持现有业务逻辑不变（匹配请求、投递请求、文案、按钮状态不变）。

## 测试策略
1. 先修改现有页面测试，要求出现 `data-testid=match-score-ring`（RED）。
2. 实现组件后运行同一测试用例（GREEN）。
3. 跑前后端编译与测试全量命令。
4. 使用 CDP 打开页面进行浏览器可视化验证。

## 风险与回滚
- 风险：仅限样式与结构变更，业务风险低。
- 回滚：回退该页面组件替换即可，接口与状态逻辑不受影响。

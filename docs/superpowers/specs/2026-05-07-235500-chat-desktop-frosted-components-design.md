# 沟通页桌面端雾面组件化改版设计

## 基本信息
- 日期：2026-05-07
- 范围：用户端 `/chat` 与企业端 `/enterprise/chat` 桌面端沟通页
- 目标：从“多重边框卡片风”升级为“商务克制 + 轻活力”的雾面工作台视觉，并完成组件化拆分，保留现有业务行为

## 背景问题
当前沟通页存在以下问题：
1. 会话列表、详情头部、消息容器、输入区大量使用边框堆叠，视觉层级依赖线条而非留白与光影。
2. 聊天工作台主组件 `ChatWorkspace.tsx` 体量过大，状态、数据请求与 UI 拼接耦合，维护和迭代成本高。
3. 用户端与企业端需要统一风格，但当前样式改动点分散，难以一致演进。

## 设计目标
1. 保留业务功能和接口行为不变（会话加载、消息发送、简历预览下载、图片预览、面试通知、已读回写等）。
2. 桌面端视觉改为雾面低边框风格：减少显式边框，改用半透明背景、轻阴影、弱分隔。
3. 通过组件拆分把会话列表、详情头部、消息流、输入区、弹层预览等 UI 从主文件中拆离。
4. 用户端与企业端共用同一组件体系，仅通过角色和数据控制内容差异。

## 视觉方案（商务克制 80% + 轻活力 20%）
1. 页面背景：低对比冷色渐变，不使用纯白满屏。
2. 主面板：`backdrop-blur` + 半透明底色 + ambient shadow，桌面端弱化外边框。
3. 会话列表项：非选中态使用浅色底；选中态用轻微发光背景与左侧色条，不使用高对比描边。
4. 消息气泡：自己消息浅品牌蓝，对方消息浅中性灰；附件消息使用低对比卡片。
5. 输入区：桌面端改为“悬浮 Dock”形态，工具按钮与发送按钮采用胶囊与弱动效。
6. 活力点缀：仅在 unread badge、主按钮 hover、状态点使用高饱和强调色。

## 组件化结构
新增并重构为以下组件（目录：`frontend/src/features/chat/components/`）：
1. `ChatAvatar.tsx`：统一头像渲染与回退首字母。
2. `ChatEmojiPanel.tsx`：表情分类、分页、选择逻辑独立。
3. `ChatPreviewModal.tsx`：PDF/图片预览弹层。
4. `ChatConversationListPanel.tsx`：左侧会话搜索与列表。
5. `ChatDetailHeader.tsx`：右侧顶部人物信息与职位摘要。
6. `ChatMessageStream.tsx`：消息流（日期分组、文本/图片/简历/面试通知消息）。
7. `ChatComposerDock.tsx`：底部输入与操作区（表情、图片、简历/面试通知、发送）。
8. `chat-style.ts`：桌面端样式 token 与 class 组合（避免硬编码分散）。

`ChatWorkspace.tsx` 保留状态编排、接口请求与事件处理，并将渲染委托给上述组件。

## 兼容性与约束
1. 移动端既有布局逻辑不变：列表页/详情页分离、固定头尾与中间滚动、返回按钮行为不变。
2. 保持现有测试 data-testid 兼容，新增样式与组件层 data-testid 用于回归。
3. 不引入新的第三方 UI 库，继续使用现有 Tailwind + shadcn 组件能力。

## 测试策略
1. 先新增/更新 `chat-workspace-redesign.test.tsx` 断言（RED），验证新的雾面样式关键类与组件挂载点。
2. 再实现组件拆分与样式迁移（GREEN），跑定向测试确保通过。
3. 最后执行前端改动面完整验证：
- `npm run test:run`
- `npm run build`

## 风险与缓解
1. 风险：组件拆分后 props 传递遗漏导致行为回归。
- 缓解：保持核心交互函数在 `ChatWorkspace` 定义，组件只做展示/回调。
2. 风险：类名变更导致已有回归测试不稳定。
- 缓解：保留关键 data-testid 与必要类名，新增更稳定的语义类断言。
3. 风险：桌面视觉升级误伤移动端。
- 缓解：样式变化优先挂载在 `md:` 及桌面专用容器类。

# AI综合能力评估真实接口设计

**日期**: 2026-04-25
**范围**: 用户端能力图谱页面（`/skill-graph`）右侧“AI 综合能力评估”卡片从前端静态计算改为后端真实接口。

## 目标

为登录个人用户提供基于技能标签的综合能力评估，输出五维分数与总分：广度、深度、结构性、时效性、稀缺性。

## 设计要点

1. 新增后端接口 `GET /person/ability-assessment`。
2. 评估输入仅使用当前用户的技能标签（来源与 `/person/graph` 相同）。
3. 固定维度与权重：
   - 广度 25%
   - 深度 25%
   - 结构性 20%
   - 时效性 15%
   - 稀缺性 15%
4. 返回结构包含：总分、各维度分、等级、标签总数、更新时间。
5. 前端能力图谱页改为调用该接口渲染评估卡，不再使用 `skills.length * 10` 静态算法。

## 评分口径

- 广度：按预定义技能类别覆盖率计分（语言、框架、数据库、中间件、前端、工程化）。
- 深度：按核心技术域标签密度与层级计分（同域标签越完整分越高）。
- 结构性：按跨域连接密度近似计分（覆盖类别越均衡、跨域越多分越高）。
- 时效性：按现代技术标签命中率计分（如 Spring Cloud、React、Docker、K8s 等）。
- 稀缺性：按稀缺标签权重加分（如分布式、微服务治理、消息队列、高并发相关标签）。

## 错误处理

- 无技能标签时返回全维度 0 分与等级 `LOW`。
- 图谱服务异常时接口仍返回 200，返回可计算结果（基于 fallback 图谱或空集合）。

## 测试策略

- 后端：
  - 评估服务单测（覆盖 0 技能、常见技能、稀缺技能场景）
  - 控制器单测（接口字段与权重输出）
  - 控制器集成测试（`/person/ability-assessment` 200）
- 前端：
  - 页面测试新增评估接口 mock，断言显示来自接口的总分与维度信息

## 非目标

- 不引入 AI 大模型生成评语。
- 不做岗位相关评估（不依赖 `jobId`）。

# 用户端个人资料保存失效修复设计

## 背景
用户反馈个人资料页面保存不生效，且字段可编辑性存在缺失。

## 根因
1. 前端将“未设置性别”提交为 `0`，数据库约束仅允许 `1/2/null`，导致保存失败。
2. `expectedSalary` 在后端 PO 标记为 `exist=false`，且数据库缺少 `expected_salary` 列，导致薪资字段无法持久化。
3. 仓储更新使用 `updateById`，空值字段默认不会更新，无法清空既有值。
4. 前端页面缺少 `age` 编辑项，字段可修改性不完整。

## 设计决策
1. 前端保存时构建规范化 payload：`gender=0 -> null`，数字字段支持 `null`。
2. 后端 `PUT /person/info` 改为覆盖式写入语义，允许清空字段。
3. 仓储层更新改为显式 `UpdateWrapper.set(...)`，确保 `null` 可落库。
4. 数据层补齐 `expected_salary` 列（schema + migration）。
5. 页面新增年龄编辑框，补齐资料字段编辑能力。

## 验收要点
1. 性别选择“未设置”可保存并刷新后保持。
2. 期望薪资可填写并刷新后保持。
3. 年龄可编辑并持久化。
4. 资料保存按钮显示成功提示，刷新后数据一致。

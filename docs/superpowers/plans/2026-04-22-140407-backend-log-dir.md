# Backend Log Directory Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task. It will decide whether each batch should run in parallel or serial subagent mode and will pass only task-local context to each subagent. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 通过 `logback-spring.xml` 将后端日志统一输出到 `backend/logs`，并保留控制台日志。

**Architecture:** 新增集中式 Logback 配置文件，定义控制台与滚动文件双 appender。通过测试先失败后通过，确保日志路径与归档策略可验证。

**Tech Stack:** Spring Boot 3.4、Logback、JUnit 5、Maven

- [ ] **Step 1: 新增最小实现**

```xml
<configuration>
  <property name="LOG_DIR" value="./logs"/>
  <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">...</appender>
  <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">...</appender>
  <root level="INFO">
    <appender-ref ref="CONSOLE"/>
    <appender-ref ref="FILE"/>
  </root>
</configuration>
```

# 验证码发送链路稳定性验收标准

## AC-1 快速返回
- Given 注册页面点击“获取验证码”
- When 调用 `/auth/send-verify-code`
- Then 接口应快速返回，不应长时间阻塞等待邮件发送结果

## AC-2 无误判失败
- Given 邮件发送链路出现慢响应或异步异常
- When 用户发起获取验证码
- Then 接口不应立即向前端返回“验证码发送失败，请检查邮箱地址或稍后重试”误判信息

## AC-3 冷却期体验
- Given 同一邮箱同一类型在 60 秒内重复点击
- When 再次请求验证码
- Then 后端返回成功（静默），且不重复发送邮件

## AC-4 限流保护
- Given 同一邮箱同一类型在 1 小时内请求次数超过 10 次
- When 再次请求
- Then 返回“验证码发送过于频繁，请1小时后再试”

## AC-5 异常回滚
- Given 后台异步发信失败
- When 任务执行失败
- Then 清理验证码与冷却/计数状态，避免用户被误限流

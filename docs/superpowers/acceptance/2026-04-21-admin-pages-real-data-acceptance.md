# Acceptance Criteria: 管理端真实数据接入与后端能力补齐

**Spec:** `docs/superpowers/specs/2026-04-21-admin-pages-real-data-design.md`
**Date:** 2026-04-21
**Status:** Approved

---

## Criteria

| ID | Description | Test Type | Preconditions | Expected Result |
|----|-------------|-----------|---------------|-----------------|
| AC-001 | 打开 `/admin/dashboard` 时页面通过正式管理端接口加载统计卡片数据而不是硬编码业务数字。 | UI interaction | 前后端服务已启动，使用管理员账号登录。 | 页面加载后可见真实统计卡片，网络请求命中 `/admin/dashboard/stats`，源码中不再依赖页面内 mock 列表作为主数据源。 |
| AC-002 | 仪表盘趋势区域仅基于真实接口返回的数据渲染，接口无数据时展示明确空态。 | UI interaction | 后端提供趋势数据或返回空列表。 | 有数据时页面显示与接口一致的趋势点位；无数据时显示空态提示，不出现静态伪趋势文案。 |
| AC-003 | 仪表盘待办事项与快捷入口数量来自真实待处理数据汇总。 | API | 后端存在至少一种待处理数据（待审核企业或失败任务）或返回零值。 | `GET /admin/dashboard/stats` 返回待办相关字段，页面显示的待办数量与接口字段一致。 |
| AC-004 | 企业审核页通过正式分页接口加载列表，并支持状态筛选与关键词搜索。 | UI interaction | 存在至少 1 条企业认证数据。 | 页面请求 `/admin/company/auth-list` 时带上 `page/pageSize/status/keyword` 查询参数，列表、总数和分页信息与响应一致。 |
| AC-005 | 管理员可在企业审核页单条通过企业认证。 | UI interaction | 存在状态为待审核的企业记录。 | 点击“通过”后调用 `PUT /admin/company/auth/{id}`，操作成功后该行状态更新为“已通过”，并出现成功反馈。 |
| AC-006 | 管理员可在企业审核页填写拒绝理由后拒绝企业认证。 | UI interaction | 存在状态为待审核的企业记录。 | 点击“拒绝”后必须提交非空拒绝理由，请求成功后该行状态更新为“已拒绝”，并保留真实拒绝理由展示。 |
| AC-007 | 管理员可在企业审核页执行批量通过或批量拒绝。 | API | 至少选择 2 条企业记录。 | `POST /admin/company/batch/approve` 和 `POST /admin/company/batch/reject` 可成功处理选中 ID 列表，返回成功后列表刷新且状态与批量结果一致。 |
| AC-008 | 用户管理页通过正式分页接口加载用户列表，并支持关键词、类型和状态筛选。 | UI interaction | 存在至少 1 条用户数据。 | 页面请求 `/admin/user/list` 时携带筛选参数，展示字段与响应一致，分页信息正确。 |
| AC-009 | 用户管理页单个用户状态修改走正式接口，不再依赖本地假切换。 | UI interaction | 至少存在 1 条可修改状态的用户数据。 | 点击启用或禁用后调用 `PUT /admin/user/{id}/status`，成功后界面状态标签刷新为接口最新值。 |
| AC-010 | 用户管理页支持批量禁用。 | UI interaction | 至少选择 2 个处于可禁用状态的用户。 | 点击批量禁用后调用 `POST /admin/user/batch/disable`，成功后选中用户状态统一更新为禁用，并清空已选计数。 |
| AC-011 | 若“批量导出”后端未实现，用户管理页明确展示为禁用或暂未开放。 | UI interaction | 打开用户管理页。 | 页面上不会出现假成功导出行为，按钮要么不可点击，要么点击仅提示暂未开放且不触发虚假下载。 |
| AC-012 | 技能标签页主列表通过真实接口加载，并支持关键词与分类筛选。 | UI interaction | 后端存在至少 1 条技能标签数据。 | 页面请求 `/admin/skill/list` 时携带 `page/pageSize/category/keyword` 参数，列表名称、分类、同义词数量和热度映射真实响应。 |
| AC-013 | 技能标签页右侧运营统计仅展示真实可得数据，无法真实提供的模块降级为空态或隐藏。 | UI interaction | 后端返回技能统计或不返回统计。 | 页面只显示真实统计字段；无真实来源时显示空态/隐藏，不出现硬编码“14,289”“342”等固定业务数字。 |
| AC-014 | 任务监控页通过正式接口加载 summary 与任务列表。 | UI interaction | 后端存在任务数据或空结果。 | 页面请求 `/admin/task/list` 后，顶部四个状态卡片与列表数据均来自响应，空结果时显示空态。 |
| AC-015 | 管理员可对失败任务执行单项重试。 | UI interaction | 至少存在 1 条状态为失败的任务。 | 点击“立即重试”后调用 `POST /admin/task/{id}/retry`，成功后该任务状态变为待处理或处理中，并出现成功反馈。 |
| AC-016 | 管理员可在任务监控页执行批量重试。 | API | 至少存在 2 条状态为失败的任务。 | `POST /admin/task/batch/retry` 成功处理选中的失败任务 ID，随后列表刷新，失败数量减少或任务状态更新。 |
| AC-017 | 管理端前后端接口路径、参数与分页结构保持一致，不再存在前端请求不存在路径或参数格式不一致的问题。 | Logic | 完成管理端 API 封装与后端 controller 改造。 | 前端 `frontend/src/lib/api/admin.ts` 中使用的所有管理端接口均能被后端对应路由成功处理，集成测试全部通过。 |
| AC-018 | 管理端五个页面在源码中移除业务假数据主渲染源，并补齐 loading、empty、error 状态。 | Logic | 完成页面改造。 | 对五个页面源码检索后，不存在用于主列表/主卡片渲染的硬编码业务数组；页面测试可验证 loading、空态和错误态。 |
| AC-019 | 后端管理端增强接口具备覆盖关键行为的自动化测试。 | API | 后端测试环境可运行。 | `mvn test` 中管理端相关测试通过，覆盖 dashboard、企业审核、用户管理、技能标签、任务重试与批量操作。 |
| AC-020 | 前端管理端页面和 API 行为具备自动化测试。 | Logic | 前端测试环境可运行。 | `npm run test:run` 中管理端相关页面/API 测试通过，覆盖真实数据渲染、筛选、错误态和关键操作。 |
| AC-021 | 管理端改造完成后，前端构建和后端编译通过。 | Logic | 前后端依赖已安装。 | `frontend` 执行 `npm run build` 成功，`backend` 执行 `mvn compile` 成功。 |
| AC-022 | 管理端五个目标页面通过浏览器进行真实联调验收。 | UI interaction | 前后端服务已启动并可在浏览器访问。 | 使用 `/web-access` skill 通过浏览器访问五个页面，确认页面展示真实数据、关键操作可用、无主流程阻塞错误。 |

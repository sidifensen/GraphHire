# Acceptance Criteria: 用户端真实数据全量接入与正式接口补齐

**Spec:** `docs/superpowers/specs/2026-04-21-user-pages-real-data-design.md`
**Date:** 2026-04-21
**Status:** Approved

---

## Criteria

| ID | Description | Test Type | Preconditions | Expected Result |
|----|-------------|-----------|---------------|-----------------|
| AC-001 | 首页首次加载时通过正式接口获取精选职位与热门企业数据，而不是使用页面内硬编码业务数组作为主数据源。 | UI interaction | 后端已启动，数据库中存在至少 1 条可展示职位和 1 条可展示企业数据。 | 打开 `/` 后页面出现接口返回的职位与企业内容；前端网络请求命中正式接口；源码中首页不再保留等价 `mockJobs`、`popularCompanies` 业务数组作为主渲染源。 |
| AC-002 | 首页聚合接口在无数据时返回可被前端识别的空列表结构。 | API | 后端已启动，首页相关职位和企业数据为空。 | 请求首页聚合接口返回 200；职位与企业字段为空列表或空统计结构；前端能展示空态而非崩溃。 |
| AC-003 | 公开职位列表接口支持关键词、城市、薪资、技能、排序与分页等正式筛选参数。 | API | 后端已启动，数据库中存在多条不同条件的职位数据。 | 请求 `/public/jobs` 及增强后的正式职位查询接口时，返回结果能按传入参数稳定过滤和分页；响应结构包含页面渲染所需字段。 |
| AC-004 | 职位列表页使用正式接口渲染职位卡片、分页与筛选结果。 | UI interaction | 后端已启动，数据库中存在至少 3 条职位数据。 | 打开 `/jobs` 后页面展示接口返回职位；切换筛选条件或分页时会重新请求接口并刷新列表；页面源码不再以内置 `mockJobs` 为主数据源。 |
| AC-005 | 公开企业列表接口返回企业页卡片所需的正式字段。 | API | 后端已启动，数据库中存在至少 3 条已认证企业及关联职位数据。 | 请求企业列表正式接口后，返回项至少包含企业标识、名称、描述/简介、标签或可推导标签、热招职位数、可选 Logo/匹配指标等页面渲染字段。 |
| AC-006 | 企业列表页使用正式接口渲染企业卡片与搜索结果。 | UI interaction | 后端已启动，数据库中存在可展示企业数据。 | 打开 `/companies` 后页面展示真实企业数据；搜索关键词后列表按接口返回刷新；页面源码不再以内置 `companies` 数组作为主数据源。 |
| AC-007 | 登录页与注册页继续走正式认证接口，不引入新的前端假业务数据流程。 | UI interaction | 后端已启动，可用测试账号已存在。 | 打开 `/login`、`/register` 后提交表单会调用正式认证接口；页面不通过静态演示数据伪造登录或注册成功。 |
| AC-008 | 简历管理页通过正式接口获取简历列表与解析状态。 | UI interaction | 后端已启动，数据库中存在当前用户的简历及不同解析状态数据。 | 打开 `/resume/manage` 后页面展示接口返回的简历列表与状态；当简历为空时展示空态；源码中无等价简历 mock 列表主数据源。 |
| AC-009 | 简历上传页上传文件后通过正式接口显示上传结果和后续状态。 | UI interaction | 后端已启动，存储与上传相关配置可用。 | 在 `/resume/upload` 上传文件后，页面调用正式上传接口并展示成功、失败或解析中状态；不使用前端伪造上传结果。 |
| AC-010 | 个人资料页通过正式接口查询并更新求职者资料。 | UI interaction | 后端已启动，当前用户资料已存在。 | 打开 `/profile` 时页面通过正式接口回填资料；提交修改后请求正式更新接口并在刷新后看到更新结果。 |
| AC-011 | 匹配详情接口返回匹配详情页所需的完整正式字段。 | API | 后端已启动，数据库中存在至少 1 条匹配记录及关联职位、用户、技能分析数据。 | 请求匹配详情接口返回 200；响应至少包含综合得分、岗位基本信息、技能重合/差距、分析维度、建议文案或可渲染字段。 |
| AC-012 | 匹配详情页使用正式接口渲染顶部评分、分析维度和建议内容。 | UI interaction | 后端已启动，存在可访问的匹配记录。 | 打开 `/match/[id]` 后页面展示接口返回的真实匹配详情；接口失败时显示错误态；源码不再使用静态示例分析数据作为主渲染源。 |
| AC-013 | 能力图谱接口返回图谱页所需的节点、边和分析维度正式结构。 | API | 后端已启动，数据库中存在用户技能、简历解析或匹配分析数据。 | 请求能力图谱正式接口返回 200；响应包含节点列表、边列表、节点属性和维度分析数据；空数据时返回可识别空结构。 |
| AC-014 | 能力图谱页使用正式接口渲染图谱与分析面板。 | UI interaction | 后端已启动，图谱接口返回有效数据。 | 打开 `/skill-graph` 后页面基于接口数据渲染图谱节点和分析区；页面源码不再以内置 `skills`、`dimensions` 数组作为主数据源。 |
| AC-015 | 通知列表接口支持当前用户通知查询、未读数与分类过滤。 | API | 后端已启动，数据库中存在多类型通知数据。 | 请求通知正式接口时可获取通知列表、未读数量及分类过滤结果；响应中包含通知标题、正文、类型、已读状态、时间和跳转目标等字段。 |
| AC-016 | 通知中心页使用正式接口渲染通知列表、分类切换和已读状态。 | UI interaction | 后端已启动，数据库中存在至少 3 条不同类型通知数据。 | 打开 `/notifications` 后页面展示真实通知；切换分类会按正式接口或正式筛选刷新列表；页面源码不再以内置 `notifications` 数组作为主数据源。 |
| AC-017 | 通知中心支持单条已读或全部已读的正式状态变更。 | UI interaction | 后端已启动，当前用户存在至少 1 条未读通知。 | 触发单条已读或全部已读操作后，正式接口返回成功，刷新列表或局部状态后未读数量与样式同步更新。 |
| AC-018 | 每个接入真实接口的用户端页面都具备 loading、empty、error 三种明确状态处理。 | UI interaction | 后端已启动；可通过测试桩或临时拦截制造空数据与接口失败。 | 在成功、空数据、失败三种场景下，页面分别展示正常内容、空态和错误/重试状态，不出现空白页或运行时报错。 |
| AC-019 | 用户端真实数据所需后端接口均具备对应自动化测试覆盖。 | Logic | 后端测试工程可运行。 | 运行相关后端测试命令时，新增或更新的控制器/集成测试全部通过，且测试覆盖首页、公共浏览、求职者资产、匹配图谱、通知等用户端页面依赖接口。 |
| AC-020 | 用户端真实数据接入后的前端页面和映射逻辑均具备对应自动化测试覆盖。 | Logic | 前端测试工程可运行。 | 运行相关前端测试命令时，新增或更新的页面测试、API 测试和 mapper 测试全部通过，覆盖真实接口接入后的正常态、空态和异常态。 |
| AC-021 | 数据库中存在可支撑用户端浏览与联调的真实可读数据。 | API | 数据初始化脚本或测试数据准备已执行。 | 对首页、职位、企业、简历、匹配、通知相关正式接口发起请求时，至少核心页面能返回非空可展示数据；若某页面本身允许空态，也应返回合法空结构。 |
| AC-022 | 完成后通过浏览器联调验证用户端四个页面域核心页面均能正常工作。 | UI interaction | 前后端服务已启动，可使用 `/web-access` 打开页面。 | 使用浏览器验证 `/`、`/jobs`、`/companies`、`/resume/manage`、`/profile`、`/skill-graph`、`/match/[id]`、`/notifications` 等核心页面时，页面能打开、能加载真实接口数据且无明显异常。 |

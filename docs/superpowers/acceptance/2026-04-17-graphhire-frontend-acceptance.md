| 验收项 | 标准 |

| TypeScript | 严格模式开启，无 any 类型 |
| 目录结构 | 符合设计文档中的 `frontend/src/app/` 结构 |
| ESLint/Prettier | 配置完成，commit 前自动检查 |

**验证命令：**
```bash
node --version  # >= 18.x
npm list next tailwindcss typescript
```

### 1.2 设计系统

| 验收项 | 标准 |
|--------|------|
| 主题变量 | CSS 变量覆盖设计系统色彩（--primary, --surface 等） |
| 字体配置 | Manrope (标题) + Inter (正文) 已配置 |
| 基础组件 | Button, Input, Card, Badge 组件可正常渲染 |

### 1.4 登录页 `/login`

| 验收项 | 标准 |
|--------|------|
| 布局 | 左右分栏（PC），单栏（移动端） |
| 表单字段 | 手机号/邮箱 + 密码 + 记住我 |
| 验证 | 必填项验证，格式验证 |

**验证方式：**
- [ ] 输入错误格式，显示验证错误提示

## 二、通用验收标准（所有阶段）

### 2.1 性能

| 指标 | 标准 |
|------|------|
| 首屏加载 | LCP < 2.5s |
| Lighthouse | Performance > 80 |
| 图片 | Next.js Image 优化，无裸 `<img>` |

### 2.2 响应式

| 断点 | 宽度 |
|------|------|
| 手机 | < 768px |
| 平板 | 768px - 1023px |
| PC | >= 1024px |

### 2.3 无障碍

| 验收项 | 标准 |
|--------|------|
| 语义化标签 | 使用 `<nav>`, `<main>`, `<button>` 等 |
| ARIA | 可交互元素有 aria-label |
| 键盘导航 | Tab 可聚焦，Enter 可点击 |

### 2.4 代码质量

| 验收项 | 标准 |
|--------|------|
| ESLint | 无 error（warning 可接受） |
| TypeScript | 无类型错误 |
| 组件拆分 | 单文件 < 300 行 |

# 用户端手机端 UI 原型提示词（重点前置版，可直接投喂 AI）

请为“GraphHire 图谱智聘-用户端”生成一套高保真手机端UI原型（iPhone 14 尺寸，375x812，中文界面，浅色主题，现代科技感，蓝色主色）。

## 重点要求（必须优先满足）
1. 所有页面必须使用中文（页面标题、菜单文案、按钮文案、表单提示、状态提示、错误提示全部为中文）
2. 底部 Tab 栏固定为：`首页 / 职位 / 公司 / 我的`
3. 页面需可点击跳转，输出完整页面流和跳转关系
4. 移动端单手操作优先：核心按钮采用底部吸附操作区

## 页面与跳转关系
1）首页（Tab：首页）
- 顶部品牌与搜索入口
- 顶部右上角通知图标（进入通知中心）
- 推荐职位卡片、推荐公司卡片
- 点击职位卡片 -> 职位详情
- 点击公司卡片 -> 公司详情

2）职位列表（Tab：职位）
- 搜索栏：关键词、城市
- 职位卡片列表（职位名、公司、城市、薪资、技能标签）
- 点击职位卡片 -> 职位详情

3）职位详情
- 展示职位信息、职位描述、技能要求
- 按钮1：开始智能匹配（页内展示匹配结果）
- 按钮2：立即投递（匹配后可用）
- 返回 -> 职位列表

4）公司列表（Tab：公司）
- 公司搜索栏
- 公司卡片列表（公司名、简介、在招数量）
- 点击公司卡片 -> 公司详情

5）公司详情
- 公司基础信息
- 在招职位列表
- 点击“查看职位匹配” -> 对应职位详情

6）我的（Tab：我的）
- 页面结构：
  - 上方“个人基础信息卡片”（头像、昵称、手机号/邮箱、求职状态、资料完整度）
  - 下方“菜单列表区”（列表项右侧箭头，点击后跳转）
- 菜单项与跳转：
  - 个人资料 -> 个人资料页
  - 简历管理 -> 简历管理页
  - 投递记录 -> 投递记录页
  - 我的图谱 -> 我的图谱页
  - 账号设置 -> 账号设置页

7）个人资料
- 基础信息编辑（姓名/性别/电话/邮箱/学历/学校/城市等）
- 求职意向编辑（目标城市/期望薪资）
- 上传头像
- 保存按钮（底部固定主按钮）

8）简历管理
- 简历列表（默认标记、解析状态、创建时间）
- 操作：设为默认、重新解析、预览、删除
- “上传新简历” -> 简历上传页

9）简历上传
- 上传区（PDF/DOC/DOCX）
- 进度状态：上传中/解析中/完成/失败
- 完成后“查看简历” -> 简历管理

10）投递记录
- 状态筛选（全部/待处理/已查看/面试邀请/已拒绝等）
- 投递记录卡片列表
- 点击“查看详情” -> 投递详情

11）我的图谱（从“我的”菜单进入）
- 能力图谱可视化区域
- 综合评分卡、节点统计卡
- 按钮：重置视图

12）通知中心（由顶部右上角通知图标进入）
- 分类筛选：全部/简历解析/职位推荐/投递反馈/系统通知
- 单条通知支持“标记已读”
- 顶部支持“全部标记已读”
- 通知动作：
  - 查看图谱 -> 我的图谱
  - 查看匹配 -> 职位详情
  - 查看职位 -> 职位列表

## 输出格式要求
- 输出：页面清单 + 页面跳转说明 + 关键交互标注
- 输出一套可点击原型（Prototype）

> 第一次投喂“第一部分：用户端提示词”

## 使用说明

你遇到的问题本质上不是页面清单不够细，而是一次性生成三端时，原型网站容易出现以下问题：

1. 漏页
2. 乱序
3. 只生成部分页面
4. 每个端内部的视觉风格不统一
5. 三端之间组件体系不一致

因此本版本改为 **分端生成**。每次只要求它生成一个端的完整页面，这样更容易保证：

- 同一端的颜色统一
- 同一端的布局风格统一
- 同一端的组件风格统一
- 同一端页面数量完整
- 同一端页面顺序稳定

## 全局统一规则

以下规则适用于用户端、企业端、管理端三部分。每次投喂任意一部分时，都默认必须遵守这些规则。

### 1. 产品定位

产品名称：**GraphHire 图谱智聘**

这是一个基于 AI 智能匹配与能力图谱的招聘平台，核心业务流程为：

**简历上传 -> AI 解析技能 -> 构建能力图谱 -> 智能双向匹配 -> 推荐通知**

产品气质关键词：

1. 专业
2. 智能
3. 可信赖
4. 高效率
5. 数据驱动

整体风格应体现“现代互联网招聘平台 + AI 数据产品”的结合感，不要做成普通招聘网站模板，也不要做成过于炫技的未来科幻界面。

### 2. 文案规则

- 所有页面文案必须默认使用中文
- 页面标题、导航、按钮、表单、表格、弹窗、状态、错误提示、成功提示必须使用中文
- 禁止大面积出现英文界面文案
- 如需保留英文，仅限品牌名、技术缩写、行业通用专有名词或用户上传的原始文件名

### 3. 组件统一规则

- 同一个端内，所有页面必须使用同一套设计语言
- 同一个端内，按钮样式必须统一
- 同一个端内，卡片样式必须统一
- 同一个端内，输入框样式必须统一
- 同一个端内，标签、状态徽标、表格风格必须统一
- 同一个端内，圆角、边框、阴影、间距体系必须统一
- 同一个端内，图标风格必须统一

### 4. 视觉统一规则

- 同一个端内，首页、列表页、详情页、表单页、通知页必须明显属于同一套产品界面
- 不允许同一端出现“首页像官网、后台像另一套系统、详情页又像第三套模板”的问题
- 颜色、卡片、导航、按钮、状态色、图标必须保持统一视觉血缘
- 即使页面类型不同，也必须让人一眼看出它们属于同一个产品端

### 5. 交互与状态规则

- 页面切换需要有轻量过渡感
- 按钮点击、Tab 切换、筛选、卡片 hover 需要有明确反馈
- Loading 优先使用骨架屏或局部加载状态
- Empty 状态必须有引导文案和主操作按钮
- Error 状态必须有错误说明和重试入口
- Success 状态必须有完成反馈

### 6. 输出要求

- 每次只生成当前投喂部分要求的页面，不要擅自补充其他端页面
- 必须按当前部分列出的顺序逐页生成
- 不得跳页、漏页、合并页
- 每一页都必须有真实内容模块，不能只有标题和空容器
- 页面之间要体现合理跳转关系和业务连续性
- 输出风格必须达到中高保真，可用于产品评审和 UI 深化

## 第一部分：用户端提示词

请你为我生成 **GraphHire 图谱智聘的用户端完整多页面原型**，面向求职者。你这一次只需要生成用户端，不要生成企业端，也不要生成管理端。

1. 清晰
2. 年轻
3. 可信赖
4. 智能推荐

请确保用户端所有页面都属于同一套视觉系统，并满足以下要求：

- 首页、登录、注册、简历、图谱、职位、通知页面必须使用统一界面语言
- 顶部导航、底部导航、按钮、输入框、卡片、标签、状态色必须统一
- 页面风格应现代简洁、清爽、可信赖，不要忽冷忽热、不要不同页像不同产品
- 推荐职位卡片、匹配度标签、能力图谱、通知卡片都必须延续同一套设计风格

### 用户端生成范围

你必须按以下顺序，完整生成用户端 **10 个页面**：

1. 首页
2. 登录页
3. 注册页
4. 简历上传页
5. 简历管理页
6. 个人信息页
7. 能力图谱页
8. 职位列表页
9. 匹配详情页
10. 通知中心

### 用户端输出规则

- 必须严格按上面顺序输出
- 不得跳页、漏页、合并页
- 不得输出企业端或管理端内容
- 每一页都必须有明确页名
- 每一页都必须包含真实业务模块
- 所有页面必须保持统一蓝色系和统一组件风格

### 用户端各页面要求

#### 1. 首页

页面目标：让求职者快速理解产品价值、搜索职位、看到推荐内容。

必须包含：

- 顶部导航：Logo、首页、职位、公司、能力图谱、登录/注册或用户头像
- Hero 区域：突出“AI 智能匹配 + 能力图谱”的产品价值
- 搜索区：关键词、城市、薪资等组合搜索
- 热门标签或热门技能
- 推荐职位模块，展示匹配度
- 热门公司模块
- 移动端底部导航：首页、职位、消息、我的

#### 2. 登录页

必须包含：

- 密码登录 / 验证码登录切换
- 手机号输入
- 密码或验证码输入
- 登录按钮
- 忘记密码、去注册入口
- 实时校验、错误提示、登录中状态

#### 3. 注册页

必须包含：

- 求职者 / 招聘者身份切换
- 手机号、验证码、密码、确认密码
- 协议勾选
- 招聘者额外字段：公司名称、统一社会信用代码

#### 4. 简历上传页

必须包含：

- 拖拽上传区
- 支持 PDF / Word 的提示
- 上传进度
- 文件信息展示
- 简历解析状态流：排队中、解析中、解析成功、解析失败
- 解析结果摘要：识别技能标签、置信度或结果概览
- 失败后的重试入口

#### 5. 简历管理页

必须包含：

- 简历列表
- 文件名、上传时间、解析状态、默认简历标记
- 操作：预览、删除、设为首要、重新解析
- 空状态引导上传

#### 6. 个人信息页

必须包含：

- 头像上传
- 基本信息
- 教育信息
- 求职意向
- 编辑与保存状态

#### 7. 能力图谱页

这是用户端核心差异化页面，必须重点设计。

必须包含：

- 能力图谱可视化区域
- 中心人物节点或头像
- 向外辐射的技能节点
- 技能节点点击详情
- 图谱统计信息
- 对比目标职位的模式
- 缩放、拖拽、全屏等操作感

视觉要求：

- 必须显著体现“AI 分析结果可视化”
- 不能只是简单标签堆砌
- 必须保持与首页、职位卡片一致的蓝色系设计语言

#### 8. 职位列表页

必须包含：

- 搜索栏
- 城市、薪资、技能、排序等筛选
- 职位卡片
- 公司信息、薪资、地点、经验、技能标签
- 匹配度展示
- 分页或无限滚动表现

#### 9. 匹配详情页

这是用户端第二个关键页面，必须体现 AI 价值。

必须包含：

- 候选人与职位信息对比区
- 总匹配度
- 五维度匹配分析：技能、经验、学历、地点、薪资
- AI 匹配原因说明
- 技能对比明细
- 投递按钮
- 查看公司按钮

#### 10. 通知中心

必须包含：

- 全部 / 简历解析 / 职位推荐 / 投递反馈 / 系统 分类
- 通知列表
- 未读状态
- 标记已读、删除等操作

### 用户端必须体现的业务流

1. 简历上传解析流程
2. 能力图谱构建与职位对比流程
3. 智能匹配与投递流程
4. 通知触达流程

### 用户端页面关系

- 首页 <-> 登录 <-> 注册
- 首页 -> 职位列表 -> 匹配详情 -> 投递结果
- 个人中心 -> 简历上传 / 简历管理 / 个人信息 / 能力图谱
- 通知中心可返回对应业务详情

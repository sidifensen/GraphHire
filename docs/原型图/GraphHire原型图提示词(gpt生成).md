# GraphHire 图谱智聘 - AI 原型设计提示词

## 角色定义

你是一位资深产品设计师 + 前端架构师，擅长使用 React/Tailwind CSS 设计系统级原型。你的任务是根据以下规范，生成完整的、可直接开发参考的高保真原型描述。

---

## 一、输出规范（强制执行）

### 1.1 基础要求
| 项目 | 规范 |
|------|------|
| **语言** | **强制全中文界面**：所有导航栏、菜单、按钮、标签、提示文字、页面标题、表单字段必须使用中文，禁止使用任何英文 |
| 完整性 | **必须一次性输出全部页面**，禁止分批或省略 |
| **图片规范** | **必须使用AI生成图片作为配图**：禁止使用简单字母（如"L"、"O"、公司名首字母）或纯色块占位图。所有图片（头像、插图、空状态图、场景图等）必须通过AI生成，风格统一、符合产品调性 |
| **响应式规范** | **必须实现响应式设计**：页面必须同时适配 PC端（≥1024px）、平板端（768px-1023px）、手机端（<768px）。描述中需明确说明各断点下的布局变化、组件隐藏/显示策略、触摸操作适配 |
| 顺序 | 用户端 → 企业端 → 管理端 |
| 细节度 | 每个页面包含：布局结构、真实字段、交互逻辑、跳转关系 |

### 1.2 页面结构模板
每个页面必须按以下格式输出：

```
## 页面名称 [页面ID]

### 布局结构
- 整体布局描述（如：左侧固定导航 + 右侧内容区）
- 响应式断点说明

### 组件清单
| 组件名 | 类型 | 位置 | 交互说明 |
|--------|------|------|----------|
| xxx | Button/Table/Form/Chart | 顶部/左侧/主内容区 | 点击后跳转至... |

### 数据字段
- 字段1：类型，说明，示例值
- 字段2：类型，说明，示例值

### 交互逻辑
1. 用户操作A → 系统响应 → 跳转目标
2. 用户操作B → 系统响应 → 跳转目标

### 跳转关系
- [按钮/链接名] → [目标页面ID]
```

## 二、设计系统规范

### 2.1 色彩体系
```
主色：#2563EB (蓝色，用于主要按钮、链接)
辅色：#10B981 (绿色，用于成功状态)
警告：#F59E0B (橙色，用于警告)
危险：#EF4444 (红色，用于删除、错误)
中性：#6B7280 (灰色，用于次要文字)
背景：#F3F4F6 (浅灰背景)
卡片：#FFFFFF (白色卡片)
```

### 2.2 字体规范
```
标题：text-xl ~ text-2xl, font-semibold
正文：text-sm ~ text-base, text-gray-600
标签：text-xs, text-gray-500
```

### 2.3 间距规范
```
页面内边距：p-6 (24px)
卡片间距：gap-4 (16px)
组件间距：space-y-4 (垂直16px)
```

## 三、全局组件定义

### 3.1 顶部导航栏（用户端）- 必须使用中文
```
布局：水平居中，固定顶部，高度 64px
左侧：Logo + 导航链接（首页、职位、公司、能力图谱）—— 所有文字必须是中文
右侧：
  - 未登录：登录按钮 | 注册按钮 —— 按钮文字为"登录"、"注册"
  - 已登录：头像下拉菜单（我的简历、我的投递、设置、退出）—— 所有菜单项为中文
```

**强制要求**：导航栏所有文字（菜单名、按钮、下拉选项）必须是中文，禁止出现英文如 Home、Jobs、Login、Settings 等。

### 3.2 底部 Tab 栏（用户端-移动端）- 必须使用中文
```
固定底部，4个 Tab：首页 / 职位 / 消息 / 我的
当前选中项高亮显示
所有 Tab 文字必须是中文，禁止使用英文如 Home、Jobs、Messages、Profile
```

### 3.3 左侧导航栏（企业端/管理端）- 必须使用中文
```
宽度：240px，固定左侧
结构：Logo区 + 导航菜单 + 底部用户信息
菜单项：图标 + 中文文字，选中状态高亮
所有菜单文字必须是中文，禁止出现英文如 Dashboard、Settings、Profile
```

**企业端菜单示例（必须使用中文）**：
- 控制台首页
- 企业信息
- 职位发布
- 职位管理
- 推荐简历
- 员工管理
- 通知中心

**管理端菜单示例（必须使用中文）**：
- 数据仪表盘
- 企业审核
- 用户管理
- 技能标签
- 任务监控

### 3.4 通用组件
- **搜索框**：带放大镜图标，支持关键词输入
- **筛选器**：下拉选择 + 标签筛选
- **分页器**：页码 + 每页数量选择
- **空状态**：插图 + 提示文字 + 操作按钮
- **加载状态**：Skeleton 骨架屏或 Spinner

## 八、关键页面详细规格

### 8.1 能力图谱页 [U07] - 核心页面

**布局结构**：
- 顶部：页面标题 + "匹配职位"按钮
- 左侧：技能图谱可视化区域（占70%）
- 右侧：技能详情面板（占30%）

**可视化要求**：
- 中心节点：用户头像
- 一级节点：技能分类（如：前端、后端、工具）
- 二级节点：具体技能（如：React、Java）
- 连线：不同颜色表示掌握程度
- 交互：支持拖拽、缩放、点击查看详情

**技能详情面板**：
- 技能名称
- 掌握程度：进度条（0-100%）
- 关联经验：项目/工作年限
- 相关职位推荐数量

---

### 8.2 匹配详情页 [U09] - 核心页面

**布局结构**：
- 顶部：候选人信息 vs 职位信息 对比栏
- 中部左侧：五维度雷达图
- 中部右侧：AI匹配原因文字说明
- 底部：技能对比表格 + 操作按钮

**五维度雷达图**：
- 技能匹配度
- 经验匹配度
- 薪资匹配度
- 地点匹配度
- 学历匹配度

**技能对比表**：
| 技能 | 要求等级 | 我的等级 | 匹配状态 |
|------|----------|----------|----------|
| Java | 精通 | 精通 | ✓ 匹配 |
| React | 熟练 | 了解 | ✗ 不匹配 |

---

### 8.3 推荐简历页 [C07] - 核心页面

**布局结构**：
- 顶部：职位选择器 + 匹配度筛选滑块
- 主体：简历卡片网格布局

**简历卡片内容**：
- 头像 + 姓名
- 基本信息：学历、工作年限、城市
- 匹配度分数：圆形进度条（大字号显示）
- 技能标签：最多显示5个
- 操作按钮：查看详情

# GraphHire 图谱智聘 - AI 原型设计提示词（三端拆分简版｜强制中文界面版｜蓝色主题优化版）

> 本文件用于把原型提示词拆成三个独立部分。
> 使用方式：不要一次性全部投喂。请按需复制下面某一个部分，单独投喂给原型生成网站。
> 本版重点优化：三端统一蓝色主题、强化中文界面约束、补强页面完整度要求、减少生成器跑偏。

---

## 第一部分：用户端提示词

请你为我生成 **GraphHire 图谱智聘的用户端原型**，这是一个基于 AI 智能匹配与能力图谱的招聘平台。

这一次你只生成 **用户端**，不要生成企业端，不要生成管理端。

用户端整体风格要求：

- 所有页面必须是同一套设计系统
- 所有页面必须统一使用蓝色主视觉，体现科技感、智能感、轻盈感
- 首页、登录页、注册页、简历相关页面、职位页面、通知页面必须看起来属于同一个产品
- 按钮、卡片、输入框、标签、状态色、导航栏样式必须统一
- 页面层级要完整，不能只给低保真线框，不能只放标题和几个空白框
- 页面文案必须使用中文
- 所有页面界面必须是中文，不允许生成英文界面
- 导航、标题、按钮、表单、表格、提示语、状态文案、弹窗文案、空状态文案、错误提示文案、成功提示文案必须全部使用中文
- 允许使用少量浅蓝、深蓝、蓝灰作为层级辅助色，但主色必须是蓝色
- 整体要像真实可演示的产品原型，不要生成通用英文模板风页面

请按下面顺序生成用户端页面：

1. 首页
2. 登录页
3. 注册页
4. 简历上传页
5. 简历管理页
6. 个人信息页
7. 能力图谱页
8. 职位列表页
9. 匹配详情页
10. 通知中心

每一页都要有真实模块，不要只写标题。每一页都必须包含合理的按钮、状态、提示、筛选或操作区，不允许生成空白占位块。

用户端页面要求如下：

### 1. 首页

- 顶部导航：Logo、首页、职位、公司、能力图谱、登录/注册或用户头像
- Hero 区域：突出 AI 智能匹配 + 能力图谱
- 搜索区：关键词、城市、薪资
- 热门标签或热门技能
- 推荐职位模块，展示匹配度
- 热门公司模块
- 产品价值说明区，体现智能解析、图谱分析、精准匹配
- 移动端底部导航：首页、职位、消息、我的

### 2. 登录页

- 密码登录 / 验证码登录切换
- 手机号输入
- 密码或验证码输入
- 登录按钮
- 忘记密码、去注册入口
- 错误提示、登录中状态
- 登录成功后的跳转提示

### 3. 注册页

- 求职者 / 招聘者身份切换
- 手机号、验证码、密码、确认密码
- 协议勾选
- 招聘者额外字段：公司名称、统一社会信用代码
- 注册成功提示或下一步引导

### 4. 简历上传页

- 拖拽上传区
- 支持 PDF / Word 提示
- 上传进度
- 文件信息展示
- 简历解析状态：排队中、解析中、解析成功、解析失败
- 解析结果摘要
- 失败重试入口
- 返回简历管理或继续完善资料入口

### 5. 简历管理页

- 简历列表
- 文件名、上传时间、解析状态、默认简历标记
- 操作：预览、删除、设为首要、重新解析
- 空状态引导上传
- 简历数量或默认简历说明

### 6. 个人信息页

- 头像上传
- 基本信息
- 教育信息
- 求职意向
- 编辑与保存状态
- 表单校验提示

### 7. 能力图谱页

- 能力图谱可视化区域
- 中心人物节点或头像
- 技能节点
- 点击技能节点查看详情
- 图谱统计信息
- 对比目标职位模式
- 缩放、拖拽、全屏操作
- 技能短板或提升建议模块

### 8. 职位列表页

- 搜索栏
- 城市、薪资、技能、排序筛选
- 职位卡片
- 公司信息、薪资、地点、经验、技能标签
- 匹配度展示
- 分页或无限滚动
- 空结果提示与重置筛选入口

### 9. 匹配详情页

- 候选人与职位信息对比区
- 总匹配度
- 五维度匹配分析：技能、经验、学历、地点、薪资
- AI 匹配原因说明
- 技能对比明细
- 投递按钮
- 查看公司按钮
- 不匹配项提醒与优化建议

### 10. 通知中心

- 全部 / 简历解析 / 职位推荐 / 投递反馈 / 系统 分类
- 通知列表
- 未读状态
- 标记已读、删除
- 空状态提示

用户端必须体现这些流程：

- 简历上传解析流程
- 能力图谱构建与职位对比流程
- 智能匹配与投递流程
- 通知触达流程

额外输出约束：

- 不要生成英文按钮、英文表头、英文导航、英文弹窗
- 不要只生成单列长页面，要体现真实产品布局
- 不要省略关键操作按钮、状态提示、空状态、异常状态
- 不要把所有页面做成同一种模板套壳，要体现用户端产品感

- 控制台、认证、职位、候选人、员工、通知页面必须看起来属于同一个产品
- 按钮、卡片、表单、表格、标签、状态色、导航样式必须统一
- 页面层级要完整，不能只给低保真线框，不能只放标题和几个空白框
- 页面文案必须使用中文

- 导航、标题、按钮、表单、表格、提示语、状态文案、弹窗文案、空状态文案、错误提示文案、成功提示文案必须全部使用中文
- 允许使用浅蓝、深蓝、蓝灰作为层次区分，但整体主色必须是蓝色

请按下面顺序生成企业端页面：

1. 企业控制台首页
2. 企业登录 / 注册页
3. 企业信息与认证页

6. 推荐简历页

8. 员工管理页

企业端页面要求如下：

### 1. 企业控制台首页

- 欢迎语
- 待处理投递数、新匹配候选人数、在招职位数
- 近期发布职位列表
- 快捷入口：发布职位、查看推荐简历、处理投递

### 2. 企业登录 / 注册页

- 企业身份明确
- 企业注册所需公司信息字段
- 登录 / 注册切换
- 错误提示和状态反馈
- 注册成功后的认证引导

### 3. 企业信息与认证页

- 企业 Logo
- 公司基础信息
- 联系人、联系方式、官网、简介
- 营业执照上传
- 认证状态：待审核、已认证、已拒绝
- 拒绝原因与重新提交入口
- 材料上传说明与审核提示

### 4. 职位发布页

- 职位名称、部门、招聘人数、地点
- 薪资范围
- 必填技能、加分技能
- 职位描述
- 保存草稿、直接发布
- 已认证企业才可发布职位的状态提示
- 表单校验与发布成功提示

### 5. 职位管理页

- 全部 / 草稿 / 已发布 / 已关闭 Tab
- 职位列表

### 6. 推荐简历页

- 按职位筛选
- 按匹配度筛选
- 候选人卡片
- 学历、城市、技能标签、匹配度
- 查看详情、不合适
- 邀请面试快捷操作

### 7. 候选人详情页

- 候选人信息与职位对比
- 五维度匹配分析
- 技能匹配详情
- AI 推荐理由
- 简历预览区域
- 邀请面试、查看更多简历、标记不合适
- 沟通记录或处理记录模块

### 8. 员工管理页

- 员工列表
- 角色区分：企业主、HR、招聘专员
- 添加员工表单
- 编辑权限入口
- 启用、停用、重置密码等管理操作

### 9. 企业通知中心

- 全部 / 候选人推荐 / 简历被查看 / 认证结果 / 系统
- 通知列表
- 未读态与已读态
- 标记已读、删除、批量处理

企业端必须体现这些流程：

- 企业认证流程
- 职位发布与职位管理流程
- 推荐候选人与候选人筛选流程
- 邀请面试与通知流程

额外输出约束：

- 不要生成英文按钮、英文表头、英文导航、英文弹窗
- 不要只生成概念化工作台，要有真实业务模块
- 不要省略认证状态、发布状态、通知状态、空状态、异常状态
- 不要把所有页面做成完全同质化卡片堆砌，要体现企业端工作效率和管理感

## 第三部分：管理端提示词

请你为我生成 **GraphHire 图谱智聘的管理端原型**，这是一个基于 AI 智能匹配与能力图谱的招聘平台。

这一次你只生成 **管理端**，不要生成用户端，不要生成企业端。

管理端整体风格要求：

- 所有页面必须是同一套设计系统
- 所有页面必须统一使用蓝色主视觉，体现秩序感、平台治理感、数据运营感
- 蓝色为主色，搭配中性色作为信息层级辅助，不要使用中性灰作为主视觉
- 仪表盘、审核、用户、标签、任务页面必须看起来属于同一个产品
- 按钮、数据卡片、筛选器、表格、标签、状态色、侧边导航样式必须统一
- 页面层级要完整，不能只给低保真线框，不能只放标题和几个空白框
- 页面文案必须使用中文
- 所有页面界面必须是中文，不允许生成英文界面
- 导航、标题、按钮、表单、表格、提示语、状态文案、弹窗文案、空状态文案、错误提示文案、成功提示文案必须全部使用中文
- 整体必须像真实平台管理后台，不要生成通用英文后台模板风页面

请按下面顺序生成管理端页面：

1. 管理端仪表盘
2. 企业审核页
3. 用户管理页
4. 技能标签管理页
5. 任务监控页

每一页都要有真实模块，不要只写标题。每一页都必须包含合理的筛选、操作、状态、数据展示模块，不允许生成空白占位块。

管理端页面要求如下：

### 1. 管理端仪表盘

- 平台核心数据卡片：用户数、企业数、简历数、职位数、匹配数
- 近 30 天趋势图
- 待办事项：待审核企业、失败任务数
- 任务成功率、活跃用户数、企业新增数等运营指标
- 快捷入口：去审核、去处理失败任务、去标签治理

### 2. 企业审核页

- 全部 / 待审核 / 已通过 / 已拒绝 筛选
- 企业信息列表
- 认证材料查看入口
- 通过 / 拒绝操作
- 拒绝理由填写或展示
- 审核详情抽屉或弹窗

### 3. 用户管理页

- 用户类型筛选
- 状态筛选
- 用户列表
- 启用、禁用、解锁
- 搜索、批量操作、状态标签

### 4. 技能标签管理页

- 标签列表
- 分类、同义词数、使用次数
- 添加 / 编辑标签
- 同义词管理
- 合并标签、停用标签、查看引用场景

### 5. 任务监控页

- 待处理、处理中、成功、失败统计
- 任务列表
- 简历解析 / 职位解析等任务类型
- 重试次数
- 失败任务高亮
- 重试操作
- 失败原因查看、筛选与批量处理

管理端必须体现这些流程：

- 企业审核流程
- 用户管理流程
- 技能标签治理流程
- 任务监控与失败重试流程

额外输出约束：

- 不要生成英文按钮、英文表头、英文导航、英文弹窗
- 不要只生成纯表格后台，要有数据卡片、状态模块、处理入口
- 不要省略审核动作、失败处理动作、批量操作、空状态、异常状态
- 不要把管理端做成单调灰色后台，必须是蓝色主视觉的现代管理平台

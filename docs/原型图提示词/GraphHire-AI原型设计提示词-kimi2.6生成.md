# GraphHire 图谱智聘 - AI 原型设计提示词（Kimi 2.6 优化版）

---

## 一、项目定位

**GraphHire 图谱智聘** — 基于 AI 智能匹配与能力图谱的新一代招聘平台。

**核心业务流程**：简历上传 → AI 解析技能 → 构建能力图谱 → 智能双向匹配 → 精准推荐通知

**设计关键词**：科技感、数据可视化、智能感知、专业可信

---

## 二、三端定位与设计规范

| 端口 | 用户 | 设计风格 | 主色调 | 辅助色 |
|------|------|----------|--------|--------|
| 用户端 | 求职者 | 现代简洁、卡片式、数据可视化 | #2563EB（科技蓝）| #10B981（成功绿）、#F59E0B（警告橙）|
| 企业端 | 招聘方 | 同用户端设计体系、左侧菜单导航 | #2563EB（科技蓝）| #7C3AED（紫罗兰）、#EC4899（粉紫）|
| 管理端 | 管理员 | 灰色系、表格为主、功能导向 | #374151（石墨灰）| #6B7280（中性灰）、#EF4444（危险红）|

**通用设计原则**：
- 圆角：卡片 12px，按钮 8px，标签 4px
- 阴影：卡片使用 `0 4px 6px -1px rgba(0,0,0,0.1)`
- 间距：基础单位 8px，遵循 8px 网格系统
- 字体：Inter / Noto Sans SC，正文 14px，标题分级 16/20/24/32px

### 3.4 简历上传页

**布局**：居中上传区域，最大宽度 720px

**内容**：
- 页面标题：上传简历，让 AI 发现你的技能图谱
- 副标题：支持 PDF、Word 格式，文件大小不超过 10MB

**上传区域**（虚线边框，拖拽高亮）：
```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│                    📄 或 🖱️ 拖拽文件到此处              │
│                                                         │
│              点击上传或拖拽 PDF/Word 文件               │
│                                                         │
│              支持格式：.pdf .doc .docx                  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**上传后状态流转**：

1. **上传中**：
   - 进度条（蓝色，带百分比）
   - 文件名 + 文件大小
   - 取消按钮

2. **解析中**：
   - AI 解析动画（脉冲效果 + 旋转图标）
   - 文案："AI 正在分析你的技能图谱..."
   - 预计时间：约 10-30 秒

3. **解析成功**：
   - 成功图标（绿色 ✓）
   - 提取的技能标签（带置信度百分比）
   - 示例：
     - Java 95%
     - Spring Boot 88%
     - MySQL 82%
     - Redis 75%
   - 操作按钮：查看完整图谱 / 继续上传 / 完成

4. **解析失败**：
   - 错误图标（红色 ✗）
   - 失败原因（文件损坏/格式不支持/内容无法识别）
   - 操作按钮：重新上传 / 手动输入技能

### 3.7 能力图谱页

**布局**：全屏可视化画布（可缩放/平移）

**核心可视化**：
```
                    ┌─────────────┐
                    │   数据分析   │

                    ┌──────┴──────┐

                    │    88%      │
                    └─────────────┘
```

**视觉规范**：
- 中心节点：用户头像（64px）+ 姓名 + 岗位标签
- 技能节点：圆形或圆角矩形，大小与掌握程度成正比
- 连线：贝塞尔曲线，粗细表示关联强度
- 颜色编码：
  - 前端技能：#3B82F6（蓝色系）

- 鼠标滚轮：缩放画布（0.5x - 3x）

  - 相关项目/经验摘要
  - 相关课程/证书

- 五维度雷达图：
  ```
        技能匹配
           ▲
          /│\
         / │ \
   经验 ◄──┼──► 薪资
  匹配    │    匹配
         / \
        /   \
   地点 ◄────► 学历
      匹配   匹配
  ```
- 各维度得分：
  - 技能匹配：85%（您掌握 8/10 项要求技能）
  - 经验匹配：90%（5年经验 vs 要求3-5年）
  - 薪资匹配：80%（期望 25k 在范围 20k-35k 内）
  - 地点匹配：100%（期望城市包含职位地点）
  - 学历匹配：100%（本科 vs 要求本科及以上）

  > "您的 Java 和 Spring Boot 技能与该职位高度匹配，5年开发经验符合要求，期望薪资在职位预算范围内。建议突出您在微服务架构方面的经验。"

- 操作按钮：

## 六、核心交互流程

### 6.1 简历上传解析流程

```
用户操作          系统反馈
─────────        ─────────
选择文件    →    校验格式/大小
                 ↓
拖拽/点击上传  →  进度条 0-100%
                 ↓
上传完成      →  AI 解析中（loading 动画）
                 ↓
解析完成      →  成功：展示技能标签+置信度
              →  失败：错误提示+重试按钮
```

### 6.2 能力图谱交互

```
初始状态：自动布局，技能节点围绕中心头像分布
    ↓
鼠标悬停节点：节点放大，连线高亮
    ↓
点击节点：弹出详情浮层（技能名/掌握程度/相关经验）
    ↓
拖拽节点：调整位置，自动保存布局
    ↓
对比模式：选择职位后，匹配技能绿色高亮，缺失技能红色标注
    ↓
缩放/平移：鼠标滚轮缩放，拖拽空白处平移
```

### 6.3 五维度匹配展示

```
雷达图：五个维度（技能/经验/薪资/地点/学历）
    ↓
各维度得分：0-100%，鼠标悬停显示详细说明
    ↓
综合匹配度：加权计算，大数字展示
    ↓
AI 匹配原因：自然语言描述匹配亮点和建议
    ↓
技能对比表：要求技能 vs 实际掌握，匹配状态可视化
```

### 6.4 投递流程

```
用户点击"投递"
    ↓
确认弹窗：选择投递简历 + 确认信息
    ↓
调用匹配接口，计算匹配度
    ↓
创建匹配记录（待处理状态）
    ↓
企业收到通知（新投递提醒）
    ↓
用户收到确认通知（投递成功）
    ↓
企业查看后，用户收到反馈通知
```

### 6.5 企业认证流程

```
企业提交认证材料
    ↓
状态：待审核（企业端显示）
    ↓
管理员审核
    ↓
    ├─→ 通过：状态变为已认证，企业可发布职位
    └─→ 拒绝：显示拒绝原因，企业可修改后重新提交
```

### 6.6 通知推送机制

| 触发场景 | 通知对象 | 通知内容 |
|---------|---------|---------|
| 简历解析完成 | 用户 | 您的简历已解析完成，共识别出X项技能 |
| 职位推荐 | 用户 | 根据您的技能图谱，为您推荐X个职位 |
| 投递成功 | 用户 | 您已成功投递"XX职位" |
| 简历被查看 | 用户 | 您投递的"XX职位"已被企业查看 |
| 新候选人推荐 | 企业 | 为您推荐X位匹配度90%+的候选人 |
| 新投递 | 企业 | 您发布的"XX职位"收到新投递 |
| 认证结果 | 企业 | 您的企业认证已通过/被拒绝 |
